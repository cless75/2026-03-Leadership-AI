## 12 категорий чувствительных данных

| # | Категория | Что искать | Действие |
|---|---|---|---|
| 1 | **Личные имена** (коллеги, руководители, родственники, клиенты) | Заглавные имена в значимом контексте | Маскирование на роль |
| 2 | **Названия компаний и продуктов** | Капитализация, словарь корпоративных названий | Замена на тип / отрасль / размер |
| 3 | **Финансы** | Числа ≥6 цифр, проценты, валюты, FY-отчётность | Обобщение или удаление |
| 4 | **Клиенты** | Имена клиентов, договоры, объёмы, даты | Замена на «клиент A» / «крупный клиент» / удаление |
| 5 | **Стратегические решения** | M&A, IPO, выход на рынок, закрытие направления, привлечение инвестиций | Review при confirmed — обязательное обсуждение с участником |
| 6 | **Кадровые решения** (увольнения, повышения с именами) | Имя + действие (уволен / повышен / снят) | Обобщение «кадровое решение прошло» или удаление |
| 7 | **Цитаты с компроматом** | Кавычки + смыслы вреда репутации цитируемого | Переформулировать или удалить |
| 8 | **Медицинские данные** (свои и других) | Диагнозы, лекарства, симптомы, тяжёлые физиологические маркеры | Удаление (не маскирование) |
| 9 | **PII** (контакты) | Email, телефон, адрес, паспорт, СНИЛС, ID документов | Удаление |
| 10 | **Юридические детали** | Договоры, NDA, условия найма, иски, споры | Удаление |
| 11 | **IP** (интеллектуальная собственность компании) | Архитектура продукта, исходный код, секретные алгоритмы | Удаление |
| 12 | **Геолокация** (специфичная) | Адреса офисов, домашние, маршруты | Обобщение «крупный город X» или удаление |

## Шаблонный mapping-словарь (роли)

Используется на Шаге 4 для предложения автозамен:

| Что | Замена |
|---|---|
| Имя CEO / основателя | СЕО |
| Имя СОО / СЕО-2 / зам. СЕО | СОО |
| Имя СРО / технического директора | СРО |
| Имя CPO / директора по продукту | CPO |
| Имя CFO | CFO |
| Имя HRD | HRD |
| Имя непосредственного руководителя | формальный руководитель |
| Имя функционального руководителя (PO, owner смежного направления) | функциональный руководитель / PO смежного продукта |
| Имя бывшего руководителя | бывший непосредственный руководитель |
| Имя коллеги (peer, тот же уровень) | линейный коллега |
| Имя коллеги из смежной команды | коллега из смежной команды |
| Имя подчинённого | подчинённый / разработчик / тимлид |
| Имя клиента (физлицо или компания) | клиент A (большой/средний/малый) |
| Имя ментора курса | ментор курса |
| Имя терапевта | терапевт |
| Имя супруга / родственника | партнёр / родственник |

Замены **предлагаются**, участник подтверждает или меняет.

## Триггерные фразы коммерческой тайны

Появление одной из следующих формулировок (или их парафраз) — повод для review:

- «выручка», «оборот», «маржа», «прибыль» + конкретные цифры
- «зарплата», «компенсация», «бонус» + цифры или имена
- «увольнение», «снятие с должности», «продвижение» + имя
- «договор», «контракт» + клиент или цифры
- «привлечение инвестиций», «раунд», «оценка компании»
- «M&A», «slияние», «поглощение», «продажа»
- «закрытие направления / продукта»
- «выход на рынок X», «уход с рынка Y»
- «технология», «алгоритм», «архитектура» + детали
- «жалоба», «иск», «суд»
- «коррупция», «откат», «откатные схемы»
- цитаты типа «врать надо…», «увольнять надо…», «не платить за…»

≥3 фраз в одном файле → confirmed → обязательный двойной запрос.

## Regex-паттерны для auto-detect

| Паттерн | Что находит |
|---|---|
| `\b[А-ЯЁ][а-яё]+\b(?!\s+([а-яё]\|$))` | Русские имена с заглавной (грубо) |
| `\b[A-Z][a-z]+\s+[A-Z][a-z]+\b` | Латинские имена «Имя Фамилия» |
| `\b[\w.+-]+@[\w-]+\.[\w.-]+\b` | Email |
| `\+?\d{1,3}[\s\-()]?\d{3}[\s\-()]?\d{3}[\s\-()]?\d{2,4}` | Телефон |
| `https?://\S+` | URL |
| `\b\d{6,}\b` | Числа ≥6 цифр |
| `\b\d+\s*%` | Проценты |
| `«[^»]{20,}»` | Длинные цитаты в кавычках |
| `"[^"]{20,}"` | Латинские кавычки длинные |

В реализации regex-проверки — приближённые; финальные решения за участником на Шаге 4.

## Уровни обезличивания и их влияние

| Цель | Имена | Компании | Финансы | Цитаты | IP | Доп. правила |
|---|---|---|---|---|---|---|
| Ментор курса | Замена на роли | Замена на типы | Обобщить | Оставить (с одобрения) | Удалить | Минимум |
| Супервизор | Замена | Замена | Обобщить или удалить | Переформулировать | Удалить | Имена внутренних обезличены |
| Терапевт | Замена + допустимы псевдонимы | Не нужны | Удалить | Переформулировать | Удалить | Фокус на эмоции/реакции |
| Peer | Замена | Замена | Удалить | Переформулировать или удалить | Удалить | Особенно жёстко с конкурентной информацией |
| Публикация | Замена + смена контекста | Полное обобщение (отрасль/размер) | Удалить | Переформулировать | Удалить | Дополнительный ручной review автором |

## Что в выходной HTML/PDF

Структура HTML-пакета:

1. **Шапка**: «Обезличенный пакет — для <цель>, <дата>». Бейджи: **обезличено**, **handoff**.
2. **Декларация обезличивания**: «В этом пакете имена заменены на роли, конкретные финансы и кадровые решения обобщены или удалены, медицинские и PII удалены».
3. **Mapping-summary** (необязательно — по выбору участника): «Имена в этом пакете соответствуют ролям: …». Полезно ментору, чтобы понимать структуру отношений; нежелательно при публикации.
4. **TOC** обезличенных файлов.
5. **Содержимое** каждого файла обезличенно, с frontmatter (но без `contains-pii: yes` и т.п. — frontmatter упрощается до `kind` и `privacy: anonymized`).
6. **Footer**: «Этот пакет создан через mentor-handoff (504). Если вы видите неожиданное имя или конкретику — сообщите участнику».
