# Запрет псевдографики

## Правило

**Не использовать псевдографику** (ASCII-арт, box-drawing символы, текстовые «схемы» из символов) в документах, заметках и ответах.

К псевдографике относятся:
- Рамки из символов `┌ ─ ┐ │ └ ┘ ├ ┤ ┬ ┴ ┼`
- «Деревья» и цепочки из `│ ▼ ├──► └──►` и подобных
- Любые схемы, нарисованные символами в моноширинном шрифте

## Чем заменять

| Вместо псевдографики | Использовать |
|----------------------|---------------|
| Блоки/карточки в рамках | **Таблицы** (колонки: название, описание) |
| Иерархия/дерево | **Маркированные или нумерованные списки** (вложенные при необходимости) |
| Причинно-следственная цепочка | **Нумерованный список** с подпунктами и стрелками в тексте (→) |
| Карта стейкхолдеров / ролей | **Таблица** (колонки: уровень/роль, фигура, описание) |
| Простые диаграммы потоков | **Списки** с явными шагами и подшагами |

## Исключения

- Блоки кода (code fence) с реальным кодом или командами — не считаются псевдографикой.
- Excalidraw-схемы (файлы .excalidraw, .excalidraw.md, вставки в заметки) и работа с ними через skill excalidraw-analyze-update допускаются и поощряются для визуализации в Obsidian.

## Политика по схемам

- **Новые схемы не создавать в Mermaid.** Формат схем по умолчанию — Excalidraw (edraw). При добавлении новых потоков, диаграмм и визуализаций использовать Excalidraw (навык excalidraw-generate / excalidraw_builder.py или excalidraw-analyze-update).
- Существующие Mermaid-диаграммы в документах можно оставлять без изменений; при доработке документа не добавлять в них новые узлы и не создавать новые Mermaid-блоки.

## Цель

Текстовые документы остаются читаемыми в любом шрифте и при копировании; структура передаётся через разметку (таблицы, списки), а не через рисование символами.
