# Навык: подготовка сдачи ДЗ 504 (504-hw-submit-prep)

**Чеклист:** [[Student-AI-Pack/templates/hw-submission-checklist|hw-submission-checklist.md]]  
**Инструкции HW:** `Module-N/Participant/504-MN-HW-Instructions.md`.

## Когда применять

Перед отправкой ДЗ; участник указал модуль **N** (1–4).

## Два режима

Перед сбором черновика проверить, есть ли у участника **собственный протокол**:

- `My-Notes/W{N}-HW-Protocol.md` или `My-Notes/M{N}-HW-Protocol.md`
- любой файл с именем вида `*HW-Protocol*.md` в корне или `My-Notes/`

**Если свой протокол найден — режим A (валидация).** Не навязывать канонический шаблон. Прочитать протокол и проверить покрытие DoD: контекст / цель / Outcome / самодиагностика до-после / next steps / связь с концептами модуля. Указать пробелы списком, предложить дополнения — но **сохранить структуру участника**.

**Если своего протокола нет — режим B (заполнение шаблона).** Канонический путь ниже.

## Процесс (режим B — заполнение шаблона)

1. Уточнить **модуль N**. Загрузить `Module-N/Participant/504-MN-HW-Instructions.md` из vault; для модулей, где файла нет — опереться на чеклист и обзор модуля (`Module-N/Module-N-Overview.md`).
2. Пройти пункты [[Student-AI-Pack/templates/hw-submission-checklist|hw-submission-checklist]] с пользователем (да/нет / что заполнить).
3. Собрать **единый черновик markdown**: Job Story, сценарий AI/без AI, протокол по шаблону из инструкции модуля, рефлексия (экзоскелет/коляска при M1).
4. Напомнить про **два канала** (мини-статус + полный протокол), если это указано в инструкции модуля.

## Процесс (режим A — валидация своего)

1. Прочитать найденный `*HW-Protocol*.md`.
2. Прогнать по чеклисту DoD: контекст, цель/Outcome, сценарий AI/без AI (если актуален), протокол действий, рефлексия, самодиагностика до-после, next steps, связь с концептами модуля.
3. Вывести **отчёт в одно сообщение**: что покрыто ✅ / что пробел 🟡 / что критично отсутствует 🔴.
4. Предложить **точечные добавления** (не переписывать файл), дождаться решения участника.
5. После согласования — дополнить протокол в местах, указанных пользователем.

## Выход

- **Режим A:** валидированный протокол + отчёт покрытия + список добавок.
- **Режим B:** готовый текст для копирования в Vault или экспорта + список открытых пунктов, если чеклист не закрыт.
