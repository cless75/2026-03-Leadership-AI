# Схема файла `ms-profile.md`

Канонический формат сквозного профиля участника Мастерской. Все поля обязательны, если не указано иное.

## Frontmatter

```yaml
---
schema: ms-participant-profile/v1
name: <псевдоним или имя>
ParticipantId: <опционально, если есть в когорте>
CDate: YYYY-MM-DD   # дата создания профиля
LastUpdated: YYYY-MM-DD
ProfileVersion: v1  # инкрементируется при каждом существенном обновлении
StartProgram: <код первой программы, с которой вошёл в MS>  # напр. "504"
MS_Programs_Timeline:  # история программ
  - program: "504"
    start: YYYY-MM-DD
    end: YYYY-MM-DD | null
    status: completed | active | paused | pivoted
tags: [ms-participant-profile, cross-program]
---
```

## Обязательные секции

### 1. Идентификация и контекст

- Псевдоним / имя
- Текущая роль / функция
- Отрасль / продукт (обезличенно, если нужно)
- Размер команды / масштаб влияния
- Как предпочитаешь общаться (формат, канал, язык)

### 2. Стиль мышления

- Аналитический / интуитивный / смешанный
- Темп (быстрый / средний / медленный)
- Формат рефлексии (письменная / устная / визуальная)
- Как принимаешь решения (интуиция / данные / обсуждение)

### 3. Ядро ценностей

- Что явно декларируешь как важное (3–5 пунктов)
- Что встроено (не всегда видно сразу, но проявляется в поступках — заполняется в Quick Refresh / Full Synthesis)

### 4. Ресурсы и сильные стороны

- Где показываешь силу (3–5 пунктов)
- Что подтверждено внешне (обратная связь, результаты)
- Что — самооценка

### 5. Уязвимости и триггеры

**EQ-самооценка (по Гоулману, 5 компонентов, шкала 1–5):**

| Компонент | Оценка | Комментарий |
|---|---|---|
| Самосознание | N | … |
| Саморегуляция | N | … |
| Мотивация | N | … |
| Эмпатия | N | … |
| Социальные навыки | N | … |

**Фокус на 2–4 недели:** какой компонент прокачиваешь.

**Триггеры (что активирует уязвимости):** список из 3–5 пунктов.

### 6. Защитные механизмы

Как справляешься со стрессом, давлением, перегрузом (анализ / контроль / уход / юмор / …). 3–5 пунктов.

### 7. Тень (опционально)

Заполняется только если участник проходил модуль самодиагностики тени (например, M3 в 504). До этого — пустая секция или пометка «не исследовано».

- Ядерная установка (если сформулирована)
- Производные паттерны
- Hogan-кластер или другой фреймворк (если использовался)

### 8. Эмоциональный фон

Короткая картина «как ты сейчас» на момент последнего обновления. 2–3 предложения.

### 9. Отношение к власти и авторитетам

- Стиль взаимодействия с руководителями (прагматика / осторожность / бунт / зависимость)
- Отдельные значимые стейкхолдеры (обезличенно, ролями)

### 10. Мотивация программы

**Декларируемые цели на старте** (заполнено в Initial Intake):
- …

**Актуальные цели** (обновляется в Quick Refresh и Mid-Course, может радикально отличаться от декларируемых):
- …

**Скрытая мотивация** (заполняется в Mid-Course и Full Synthesis, если есть):
- …

### 11. Прогресс по программам MS

Для каждой пройденной или активной программы:

```
#### <программа>

- Старт: YYYY-MM-DD
- Статус: active | completed | paused | pivoted
- Модули пройдены: M1 ✅ M2 ✅ M3 🟡 M4 ⬜
- Ключевые сдвиги:
  - …
- Главный результат: …
- Open questions: …
```

### 12. Пропущенные checkpoints (для ведущего)

Если участник явно пропустил обязательный Mid-Course — фиксация:

```
- Программа: 504, Module: 3, Причина: <участник указал>, Дата: YYYY-MM-DD
```

## Необязательные секции

- **Внешние резюме/CV** — ссылка.
- **Поле ментора** — если есть назначенный ментор из MS.
- **Цели жизненного плана** — если участник хочет их вести в одном месте.
- **Связь с другими инструментами MS** — ссылки на Wardley Map, Disney Map, PBS и т. п., если применимы.

## Правила обновления

1. **Каждое обновление** инкрементирует `ProfileVersion` и обновляет `LastUpdated`.
2. **Старая копия** сохраняется в `ms-profile.history/ms-profile.vN.md` (номер предыдущей версии).
3. **В секции «Прогресс по программам MS»** — только добавлять, не переписывать историю.
4. **Секция «Мотивация программы»** — декларируемые цели **не меняются** (это исторический факт), добавляются «Актуальные цели» и «Скрытая мотивация».

## Минимальный валидный профиль

Для первого запуска после Initial Intake достаточно заполнить секции 1, 2, 5 (без Hogan-детали), 10 (только декларируемые цели), 11 (одна активная программа). Остальное заполняется по ходу.
