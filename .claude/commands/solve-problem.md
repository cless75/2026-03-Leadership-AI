## Входные данные

Описание проблемной ситуации: $ARGUMENTS

Если описание не указано — запроси у пользователя:
1. Название продукта / системы / организации
2. Что именно беспокоит (нежелательные явления)
3. Кто затронут (стейкхолдеры)

## Методология

Pipeline из 4 навыков курса «Системное и стратегическое мышление» (514-CM-MIPT).

**Канон навыка:** [.cursor/skills/stai-problem-solving/SKILL.md](.cursor/skills/stai-problem-solving/SKILL.md)

**ВАЖНО:** Перед началом работы прочитай SKILL.md. Следуй алгоритмам точно, не пропускай шаги.

## Порядок выполнения

### Фаза 1: Формулирование проблемы (Skill 1)

1. **Проблематизация** — заполни таблицу 5 элементов (НЯ, важность, контекст, стейкхолдеры, to-be)
2. **Формулировка gap** — As-is / To-be / Gap
3. **Классификация** — Собиратель / Охотник / Координатор (+ домен Cynefin)
4. **Диагностика типа корневой причины** — 2-3 цепочки «5 Почему» → простая / убеждения / динамика

**Диалоговая точка Д1:** Покажи результаты Фазы 1. Спроси: согласен? нужны правки?

---

### Фаза 2: Диагностика (Skill 2)

1. **Маршрутизация** по типу корневой причины (Простая → 5 Why; Убеждения → Iceberg; Динамика → CLD)
2. **Построение модели** (Iceberg и/или CLD)
3. **Распознавание архетипа** — Limits to Growth / Shifting the Burden / Fixes that Fail / Eroding Goals / Success to Successful
4. **Определение типичной точки рычага** архетипа

**Диалоговая точка Д2:** Покажи модель, архетип. Спроси: узнаёт ли пользователь свою систему?

---

### Фаза 3: Решение (Skill 3)

1. **5 уровней рычага Медоуз**
2. **Карта стейкхолдеров** — матрица влияние × интерес
3. **Приоритизация** — матрица Effort / Impact
4. **Портфель 3-5 гипотез** (Quick Wins + стратегические; минимум 1 на уровне 3+)

**Диалоговая точка Д3:** Покажи портфель. Спроси: реалистично? приоритеты? нужна Фаза 4?

---

### Фаза 4: Реализация (Skill 4) — опционально

1. **Дизайн экспериментов** — Plan → Test → Metric → Decision
2. **Грозовая туча** (при конфликте) — TOC-анализ
3. **Риск-анализ** — 3 риска × митигация
4. **Anti-hallucination check**

Стратегическое расширение (по запросу): Creative Tension, Сценарное мышление, Стратегические ставки.

## Выходные артефакты

- [ ] Формулировка проблемы (5 элементов + gap)
- [ ] Тип задачи и тип корневой причины
- [ ] Системная модель (Iceberg / CLD) + архетип
- [ ] Портфель из 3-5 гипотез с приоритизацией
- [ ] (опц.) Планы экспериментов + риск-анализ

## Режим коротких итераций

На каждом шаге показывай промежуточный результат и спрашивай подтверждение.

## Два трека решений

| Трек | Горизонт | Фокус |
|------|----------|-------|
| **A: Коррекция сейчас** | Текущий запуск | Quick wins, работа с тем что есть |
| **B: Редизайн на будущее** | Следующий запуск | Структурные изменения |

## Принципы работы

- Следуй алгоритмам из навыка точно
- На каждой диалоговой точке останавливайся и жди подтверждения
- Работай короткими итерациями
- Парадокс рычага: чем сильнее рычаг — тем сложнее использовать
- Не подменяй проблему решением
