# My-Notes — личные конспекты участника

Здесь участник ведёт **свои** заметки. Содержимое **не коммитится** в публичный репозиторий — `.gitignore` исключает всё, кроме этого `README.md` и шаблонов (`TEMPLATE-*`).

## Что куда класть

| Тип артефакта | Путь | Шаблон | Когда |
|---------------|------|--------|-------|
| **Пояснительные записки (ПЗ) комплексных задач** | `PZ/<ProjectId>-п-<Name>.md` | [[TEMPLATE-PZ]] | Длинные мыслительные процессы по проекту/защите/инициативе |
| **Недельные конспекты курса** | `<Module>-summary.md` (корень `My-Notes/`) | [[TEMPLATE-WN-summary]] | Итог по модулю/неделе курса |
| **Сессии (логи сессий с AI)** | `Sessions/<YYYY-MM-DD>-<тема>.md` | — | Длинные разговоры с AI, которые не вписываются в daily |
| **Диагностики (выгорание, EQ, профиль)** | `Diagnostics/<тип>-<YYYY-MM-DD>.md` | автомат через скиллы | Результаты `/burnout-check`, `/eq-self-diagnosis` |
| **Идеи и инсайты в течение курса** | `Ideas/<YYYY-MM-DD>-<тема>.md` | — | Сырые мысли, которые жалко терять |
| **Личное (черновики, эксперименты)** | `Private/` | — | Что не хочется показывать никому |
| **Логи (отчёты, дебаг)** | `Logs/` | — | Технические логи AI-инструментов |

## Альтернатива для ПЗ

Если хочется, чтобы ПЗ была в общей timeline проекта (рядом с daily-заметками и материалами проекта) — клади её в `2026/<тема>/ПЗ-<Name>.md`. Папка `2026/` тоже в `.gitignore`, но она **видна** в Obsidian рядом с другими проектными артефактами.

## Что НЕ класть в корень vault `504-demo/`

ПЗ, личные конспекты и сессии в корне vault попадают под git и могут случайно закоммититься. По умолчанию **всё личное → в `My-Notes/`**.

- [[TEMPLATE-WN-summary]] — недельный конспект (итог по модулю)
