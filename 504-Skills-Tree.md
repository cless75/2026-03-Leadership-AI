## Слой 2 — Мета-скиллы (для разработчика курса)

Эти скиллы используются автором курса (Дмитрий Безуглый и команда MS) при создании и поддержке материалов. Участникам не нужно их применять напрямую, но полезно знать что курс не статичен — модели и материалы эволюционируют через них.

### Жизненный цикл моделей

| Скилл | Когда применяется | Что делает |
|-------|-------------------|------------|
| `ms-create-model-card` | При появлении новой канонической модели | Создаёт карточку в `BoK/Concepts/` (или `MS-Courses/Concepts/` в редакционном репо) с обязательными секциями (Определение, Логика, Ядро, Связи, Эволюция) |
| `ms-adapt-model` | При применении канона к конкретному продукту/курсу | Создаёт адаптацию канона под контекст, фиксирует расхождения в ПЗ продукта |
| `ms-update-canon` | При эволюции модели (новые инсайты, уточнения) | Обновляет блок «Эволюция» канон-карточки, при изменении ядра — требует подтверждения автора |
| `ms-consistency-review` | Периодически или после `ms-update-canon` | Сверяет все адаптации с актуальным каноном, выявляет рассинхронизацию |

### Жизненный цикл лекций

| Скилл | Когда применяется | Что делает |
|-------|-------------------|------------|
| `ms-prep-lecture` | До проведения лекции/воркшопа | Контекст из программы, FG, актуализация плана, NotebookLM-пакет, классический и flipped run-of-show |
| `ms-process-lecture` | После проведения лекции | Обработка транскрипции, выявление новых моделей, классификация коллизий (OK / Обновление канона / Новый инсайт / Коллизия) |
| `ms-illustration-workflow` | При создании визуальных артефактов | Edraw-схемы, Illustration-Registry, привязка к моделям |
| `ms-import-slides-concepts` | При импорте концепций из Google Slides | Текст слайдов → черновики канон-карточек по шаблону MS-Courses |
| `ms-manage-collision` | При обнаружении расхождения канон ↔ адаптация | Классификация, обсуждение с автором, разрешение |

## Карта применения в защите M4

Защита M4 даёт три трека на выбор. Карта показывает какие скиллы курса (Слой 1) и общие (Слои 3-4) применять на каждом этапе.

### Подготовка → Защита → Обработка

| Этап | Трек A (лидерство) | Трек B (команда) | Трек C (миссия + позиционирование) |
|------|--------------------|------------------|-------------------------------------|
| **Подготовка** | S4.01 (рефлексия курса) → S4.06 (Primal Leadership стиль) → S0.03 (тень) | S4.01 → S2.01 (фаза Такмана) → S2.02 (роли) | S4.01 → **S4.03 (миссия)** → **S4.03e1 (BIP+CBBE)** ⚡ |
| **Защита** | Q&A: кейсы личного развития, разбор ситуаций | Q&A: групповые сценарии, командная динамика | Презентация миссии + позиционирование, обоснование квартальных действий |
| **Обработка** | S4.04 (feedback от группы) → дополнить профиль | S4.04 + S2.03 (ритуалы команды) → план команды | S4.04 → дополнить Decision Memo + Brand Refinement Memo |

### Сквозные навыки для всех треков

- **`session-link`** — START в начале каждой подготовительной сессии, CLOSE с фиксацией результата в daily-заметку
- **`micro-decision-514`** — для ключевых выборов (например выбор трека, выбор Primal-стиля, выбор квартального действия)
- **Decision Memo внутри артефактов** — фактура мышления, не теряемая при формулировании итогов
- **`burnout-self-diagnosis`** перед интенсивной фазой подготовки — зона работы, не зона выгорания
- **`/profile-refresh`** после защиты — обновить сквозной профиль учётом инсайтов M4

### Когда подключать расширение S4.03e1

S4.03e1 (Brand Identity & Equity Refinement) — опциональный углубляющий слой над S4.03. Подключай если:
- Идёшь по треку C (миссия + позиционирование)
- Миссия должна выйти на внешнюю аудиторию (клиенты, рынок, инвесторы)
- Чувствуешь разрыв между «что декларируем» и «что доходит до клиента»

Не подключай если:
- Миссия ещё не сформулирована (сначала базовый S4.03)
- Команда работает на закрытом рынке без брендовой коммуникации
- Меньше месяца до защиты — расширение требует времени
