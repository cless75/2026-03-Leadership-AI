# Домашние задания мастерской 504 — как работать

## Зачем этот гайд

ДЗ мастерской — не «галочка», а связка **практики на работе**, **рефлексии** и **прозрачности для ментора**. Этот файл даёт единый порядок действий вместе с AI (Cursor / Claude Code).

## Порядок (каждая неделя N)

1. Открой **инструкцию недели:** `Module-N/Participant/504-MN-HW-Instructions.md`  
   В этом репозитории доступны **M1** и **M2**; для M3–M4 используйте файлы, которые выдаст команда курса.
2. Прочитай **обзор недели:** [[Module-1/Module-1-Overview|Week 1]] или [[Module-2/Module-2-Overview|Week 2]] — цель, результаты, концепты.
3. Пройди **чеклист перед сдачей:** [[Student-AI-Pack/templates/hw-submission-checklist|hw-submission-checklist]].
4. Вызови в Claude Code команду **`/504-hw-submit-prep`** (или опиши в чате: «подготовь ДЗ 504, неделя N») — навык соберёт черновик по инструкции и чеклисту.

## Каналы сдачи

Конкретные каналы (например краткий статус в чат курса + полный протокол в файле) **всегда берутся из инструкции недели**. Не подменяй их общими советами.

## ПДн и обезличивание

- Не вставляй в публичные артефакты персональные данные коллег без необходимости; заменяй роли и контекст на обезличенные формулировки.
- Файлы `goals-504.md` и `my-profile.md` не предназначены для публичного коммита (см. `.gitignore`).

## Жизненный цикл профиля участника

Помимо HW по модулям, в курсе есть **сквозной цикл профиля**. Он идёт параллельно — фиксирует твою эволюцию от старта до конца программы и переносится в следующие программы MS.

| Когда | Что | Команда | Длительность |
|---|---|---|---|
| **Старт курса** (W0/W1) | Создать профиль участника Мастерской — `ms-profile.md` | `/profile-intake` | 20–30 мин |
| **Конец каждого модуля** (опционально) | Обновить профиль по дельтам модуля | `/profile-refresh` | 20–30 мин |
| **После M2 — обязательно** | Mid-Course Diagnostic: сравнение деклараций и фактов, контр-диагностика, выбор Continue/Adjust/Pivot | `/mid-course-diagnostic` | 45–60 мин |
| **После M4** | Full Synthesis: полный реверс-инжиниринг профиля + опционально рекомендации | `/profile-synthesis` | 1.5–2 ч + сутки паузы |

Профиль живёт **не в репо курса**, а в личном рабочем пространстве. Путь сохраняется в `.claude/settings.local.json` при первом `/profile-intake`. Это позволяет переносить профиль между программами MS.

**Почему Mid-Course обязательный:** после M2 часто происходит незаметный сдвиг целей. Без специального checkpoint'а программа продолжает двигать не то, что тебе нужно сейчас. HW M2 — [[Module-2/Participant/504-M2-HW-Mid-Course-Diagnostic|504-M2-HW-Mid-Course-Diagnostic]].

## Связь с ПЗ и рефлексией

По желанию зафиксируй одну строку итога ДЗ в своей пояснительной записке (`ПЗ-*.md`) в секции `## Ход работы` или в ежедневной заметке `YYYY/01-Daily/YYYY-MM-DD.md`.

## После обновления репозитория

После `git pull`, если в корне остались папки `Week-1/`, `Week-2/` — это наследие переименования `Week → Module`. Удалите их вручную:

```bash
rm -rf Week-1 Week-2
```

Актуальная структура материалов — `Module-1/` … `Module-4/`. Также можно вручную удалить default-файлы Obsidian, если они появились в корне: `Без названия.canvas`, `Добро пожаловать.md` — это локальные артефакты, не часть курса.
