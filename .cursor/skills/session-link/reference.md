## HANDOFF-PATH: файл session-handoff (чат-мост)

**Назначение:** краткое резюме для **следующего** AI-чата после **CLOSE** (по умолчанию, см. **CLOSE-5** в [SKILL.md](SKILL.md)).

**Важно:** это **не** журнал дня и **не** обязательный `ctx`.
Подробная фиксация сессии — в **`## Заметки в течение дня`**; опционально одна строка в **`## Ключевой файл сессии (ctx)`**.

| Путь | Когда использовать |
|------|-------------------|
| **`.claude/session-handoff.md`** | Единственный путь в vault 504-demo |

**Правило:** не дублировать полные тексты ПЗ; 2–3 итога, 1–3 шага, имя Objective и при наличии ПЗ.

## Формат ссылок Obsidian

Все ссылки на файлы vault — **только wikilinks** `[[path]]` или `[[path|alias]]`. Не использовать: `[text](url)`, `` `path` `` (backticks).

## Журнал сессий: `## Заметки в течение дня` (START-7b, MID-6, CLOSE-7)

Искать секцию **`## Заметки в течение дня`** в `YYYY/01-Daily/YYYY-MM-DD.md`.

### Карта статусов (в скобках заголовка `### HH:mm [[Objective]] (…)`)

| Эмодзи | Статус |
|--------|--------|
| 🔵 | Старт |
| 🟡 | Продолжение |
| 🟢 | Выполнена Objective |
| 🟣 | DoD на сегодня завершен |
| ⚪ | Отложена (на будущее) |
| 🔴 | Не сработало |

### Старт (START-7b)

```markdown
### 09:15 [[186-18 Навык управления проектами (project-management) 🔄]] (🔵 Старт)
- [ ] Начало работы · ИмяАгента (Cursor) · [[имя-заметки-для-сессий]]
```

Ценность (**START-2b**), если есть — отдельной строкой: `Ценность: согласовать границы PBS` (или `· ценность: …`).

Второй заход **по той же** Objective в тот же день:

```markdown
### 14:30 [[186-18 Навык управления проектами (project-management) 🔄]] (🟡 Продолжение)
- [ ] Начало работы · Claude (CC)
```

### Промежуточно (MID-6)

Под **открытым** заголовком (🔵 или 🟡):

```markdown
- [x] Обновлён reference session-link; согласован формат журнала
- [ ] Следующая задача: синхронизировать команды `.claude/commands`
```

### Финал (CLOSE-7)

Заменить скобки в **том же** заголовке, например:

```markdown
### 09:15 [[186-18 Навык управления проектами (project-management) 🔄]] (🟢 Выполнена Objective)
```

или `(🟣 DoD на сегодня завершен)` / `(⚪ Отложена (на будущее))` / `(🔴 Не сработало)`.

### Резюме: продолжаем vs новая задача

- **Та же Objective:** новый `### … (🟡 Продолжение)` или MID-строки под текущим открытым блоком.
- **Другая Objective:** новый `### … (🔵 Старт)`.

## Секция ctx (опционально, START-7c, CLOSE-7)

Искать **`## Ключевой файл сессии (ctx)`** или алиас **`## Сессии / ctx`**.

**Открыто:**

```markdown
- [ ] [[186-18 Навык управления проектами (project-management) 🔄]] — открыто 2026-04-02 09:15 (Cursor) · журнал: § Заметки в течение дня
```

**После CLOSE** (если строка велась):

```markdown
- [x] [[186-18 …]] — открыто 2026-04-02 09:15 (Cursor) — итог зафиксирован (Notes) · [[186-п-…]]
```

## Внеплановые задачи в дне (CLOSE-8)

В **`YYYY/01-Daily/YYYY-MM-DD.md`** под **`## Objectives на сегодня`** — подсекция **`### Внеплановые задачи`**: ручные `- [ ]` пункты. **Ссылка на задачу:** по возможности `- [ ] [[карточка Objective]]`.

---

## Строка старта сессии в Objective (шаблон)

В секции `### Сессии` или `### Работа по дням`:

```markdown
- 2026-04-03 09:15 — старт (Cursor) → [[2026-04-03]]
```

Варианты пометки среды: `(Cursor)`, `(CC)`, `(офлайн)` — по факту.

## Поиск ПЗ в проекте

- Имя: `ПЗ-*.md`, `{ProjectId}-п-*.md`.
- Корень: папка проекта из frontmatter Objective или карточки `#project`.
- Глубина: сам каталог + подпапки на 1–2 уровня.

---

## CLI (напоминание)

- **Cursor:** полный путь к файлу в vault, напр. `cursor "C:\Obs-Vaults\504-demo\path\to\file.md"`.
- **Obsidian:** URI **`obsidian://open?vault=504-demo&file=<путь-от-корня-vault>`**.
- **День «сегодня»:** `obsidian daily` или вручную `YYYY/01-Daily/YYYY-MM-DD.md`.

## Следующие шаги после CLOSE (CLOSE-4a)

После черновика итога сессии агент **обязан** задать явный вопрос: какие 1–3 следующих шага записать в Notes Objective и при необходимости в ПЗ (или «нет»). Пример блока в Objective:

```markdown
### Следующие шаги (зафиксировано 2026-04-03)
- …
- …
```
