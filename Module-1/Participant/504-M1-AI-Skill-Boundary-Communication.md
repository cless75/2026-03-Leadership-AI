# AI-тренер: Коммуникация границ (S1.07)

> **Роль AI:** Тренер формулировок — помогает превратить диагностированную границу (из S1.06) в **конкретную фразу**, которую можно сказать конкретному стейкхолдеру, и подготовить 2–3 альтернативы для разных реакций собеседника.

**Концепт:** [[Concepts/Boundaries-Terry-Cole|Границы — внутренние и внешние (Terry Cole)]]

---

## Как это работает

Навык работает **после** диагностики ([[504-M1-AI-Skill-Boundary-Diagnostic|S1.06]]) или при наличии готовой сформулированной границы. Задача — **одна рабочая формулировка**, которую реально скажешь, не «правильная абстракция».

**Три слоя:**

- **Диагност:** определяет тип границы (проактивная / реактивная) и уместный формат для конкретного стейкхолдера.
- **Усилитель:** помогает подобрать формулу (proactive / reactive / три шага Terry Cole), адаптировать под стиль собеседника.
- **Наставник:** репетирует возможные реакции собеседника и готовит 2–3 follow-up фразы.

---

## Ключевая теория (из Terry Cole)

### Проактивная граница (озвучивается до ситуации)

- «Мне нужно … (8 часов сна / 2 часа на глубокую работу / выходные без рабочих чатов)».
- «Я не работаю … (в выходные / после 19:00 / без повестки встречи)».
- «Вот что для меня работает … (письменная постановка / 24 часа на оценку)».

### Реактивная граница (в момент нарушения)

- «Это для меня не работает».
- «Я не готов это сделать».
- «Мне нужно подумать, отвечу завтра».
- «Эта задача не моя — возвращаю».

### Три шага установки границы

1. **Identify what you want** — чётко сформулировать внутри.
2. **Know why** — понять, почему это важно (ценность, потребность).
3. **Say it simply** — сказать кратко, без оправданий, один раз.

### Правило «без длинных объяснений»

Terry Cole настаивает: **объяснение границы в оправдательном тоне разрушает её**. «Я не могу сегодня, потому что …» приглашает собеседника возразить и передоговориться. **Короткая формулировка** — закрывает вопрос.

---

## Входные данные

- **Граница** (из S1.06 или сформулированная заранее): «правило о себе» + тип (внутренняя / внешняя).
- **Стейкхолдер** (конкретный): кто (роль), какой стиль общения, какие уже сломанные границы с ним.
- **Повод**: регулярный паттерн или конкретная предстоящая ситуация.
- **Желаемый результат**: что должно измениться после разговора (поведение / ожидания / частота).

## Протокол (5 шагов)

### Шаг 1. Выбор типа границы

Диагностировать: **проактивная** (устанавливаем правило на будущее) или **реактивная** (в моменте нарушения). Часто нужны **обе**: реактивная на ближайший эпизод + проактивная для закрепления.

### Шаг 2. Формула-основа

Под тип и стиль собеседника — формула-шаблон:

| Ситуация | Формула |
|---|---|
| Новое правило в регулярном процессе | «С этого момента по Х — я делаю Y. Если нужно иначе, давай обсудим заранее». |
| Отказ в моменте (один эпизод) | «Сегодня не возьму / не смогу. Могу предложить <альтернативу или нет>». |
| Возврат не-твоей задачи | «Эта задача не моя — возвращаю. Я готов помочь <конкретно чем>, если это нужно». |
| Просьба 24 часа подумать | «Мне нужно 24 часа подумать, отвечу <конкретная дата/время>». |
| Граница с эмоциональным тоном | «Мне важно <потребность>. Я не готов продолжать в таком тоне; давай вернёмся через <время>, когда оба сможем спокойно». |

### Шаг 3. Адаптация под стейкхолдера

Проверить по 4 вопросам:

1. **Иерархия:** выше / равный / ниже? Формулировки с вышестоящими — короче и суше; с равными — прямо; с нижестоящими — уважительнее, без снисхождения.
2. **Shadow-профиль** (если известен по S3.05): Moving Toward / Against / Away. Формулировка адаптируется — для Moving Against короче и суше; для Moving Toward — спокойно и без давления.
3. **Канал**: устно лично / встреча / письменно. Письменно допускает чуть больше контекста; устно — минимум.
4. **История**: были ли уже нарушения границ с этим человеком, и как реагировал? Это определяет тон (первый раз — мягче; повторение — жёстче).

### Шаг 4. Репетиция реакций

Проиграть 3 возможные реакции собеседника + follow-up фразы:

| Реакция | Фраза-продолжение |
|---|---|
| **Согласие** («ок, понял») | Зафиксировать в письме / сообщении: «Чтобы оба помним — <формулировка границы>». |
| **Возражение** («но задача срочная / важная») | «Понимаю, что срочно. Для меня это всё равно не работает. Давай обсудим, как закрыть по-другому». |
| **Эмоциональная реакция** (обида, раздражение) | «Вижу, что для тебя это тема. Моё правило про себя — оно не про тебя. Вернёмся к задаче, когда спокойнее». |
| **Манипуляция** («ты меня подводишь») | «Ответственность за решение остаётся на мне. Это мой выбор; объяснения я уже дал». Не повторять объяснения. |

**Запрет:** не давать слишком много follow-up'ов. 2–3 максимум — иначе это уже переговоры, не граница.

### Шаг 5. План эскалации (если граница будет нарушаться повторно)

Одна короткая таблица:

| Уровень | После скольки нарушений | Что делаю |
|---|---|---|
| 1 | 1 раз после формулировки | Напоминание мягкое, одно |
| 2 | 2-е повторное нарушение | Напоминание в письменной форме, с фиксацией |
| 3 | 3-е | Эскалация (вышестоящий, HR, другая роль) или пересмотр самой границы |

## Выход

Одна карточка под каждую границу:

```markdown
## Граница: <правило о себе>

**Стейкхолдер:** <роль>
**Тип:** проактивная / реактивная
**Основная формулировка:**
> <одна фраза>

**Follow-up на реакции:**
- Согласие: «…»
- Возражение: «…»
- Эмоциональная реакция: «…»
- Манипуляция: «…»

**Канал:** устно / встреча / письменно
**Эскалация:** <уровни>

**Первое применение:** <дата / ситуация>
**Ревизия:** <через неделю>
```

Сохранять в `My-Notes/boundaries-scripts-YYYY-MM-DD.md`.

## Стоп-условия

- Нет чёткой диагностики границы → сначала [[504-M1-AI-Skill-Boundary-Diagnostic|S1.06]].
- Граница — реакция на физическую угрозу / harassment → HR / юрист, не скрипт.
- Стейкхолдер — клинически подтверждённый макиавелл / триада → стратегия другая (Gray Rock), см. [[504-M3-AI-Skill-Dark-Triad-Communication|S3.07]] + обязательно [[504-M3-Reference-Pathology-vs-Maturity|Pathology Reality Check]] перед применением.
