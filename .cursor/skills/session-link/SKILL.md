## Handoff и session-link: одна цель, два уровня

**Общая цель:** зафиксиро��ать результат так, чтобы работу можно было **продолжить**, в том числе при **обрыве** сессии (краш, смена чата, новый диалог, исчерпание контекста в другом контуре).

| Уровень | Назначение | Когда |
|--------|------------|--------|
| **GTD (vault)** | Канон истины: ПЗ, `## Notes` Objective, **журнал сессий** в `## Заметки в течение дня` файла `YYYY/01-Daily/…`; опционально одна строка-индикатор в `## Ключевой файл сессии (ctx)` | START / MID / CLOSE по подпротоколам ниже |
| **Чат-мост (`session-handoff`)** | Сжатое резюме для **следующего** AI-чата без обязательного чтения всей ПЗ | **По умолчанию пос��е успешного CLOSE** (шаг **CLOSE-5**); путь — [reference.md](reference.md) |

**Жёсткое правило (канон журнала):** подробная фиксация AI-сессии в дне — в секции **`## Заметки в течени�� дня`** (заголовки `### HH:mm [[Objective]] (статус)` и пункты под ними). Секция **`## Ключевой файл сессии (ctx)`** — **опциональный** компактный индикатор (**START-7c**, **CLOSE-7**); она **не заменяет** журнал.
Файл `.claude/session-handoff.md` **не является** ни журналом, ни `ctx`; не заменяет запись в vault.

**Триггер слова «handoff»** в контексте Objective/проекта ведёт на **CLOSE** (полная GTD-фиксация). Это не освобождает от записи в vault.

## Когда применять

**START** — явный фокус на задаче в **начале** работы:

- «Работаем над …», «открой контекст проекта N», «начинаем сессию по задаче …», «session start», указан `ProjectId` / открыт файл Objective.
- В составе **«Зафиксируй сессию»**: если существенного итога ещё нет и пользователь подтверждает **только старт** — выполнять **START** (штамп + журнал **START-7b**; опционально **START-7c**), **без** обязательной строки в ПЗ «Ход работы».

**MID** — **промежуточная** фиксация: сессия **продолжается**, нужно зафиксировать состояние:

- «Промежуточный итог», «зафиксируй промежуток», «checkpoint», «сохрани состояние», «зафикси��уй сессию» + явно **«промежуточно»** / «пока не закрываем».
- ПЗ + Notes дополняются; в **`## Заметки в течение дня`** дописываются строки промежуточного итога (**MID-6**); опциональная строка **ctx** (**START-7c**) **остаётся** `- [ ]`. **CLOSE-4a не обязателен** — по желанию один короткий вопрос о шагах или пропуск.

**CLOSE** — **финал** сессии или пауза с полной записью:

- «Зафиксируй результат», «фиксируем работу», «закрываем сессию с записью», «session result», «handoff» **в контексте Objective/проекта**, пересказ итога **офлайн/устно** текстом.
- В составе **«Зафиксируй сессию»**: если есть **существенный итог** или пользователь хочет **закрыть** сессию — **CLOSE** (ПЗ, Notes, **обязательный CLOSE-4a**, **CLOSE-7** — смена статуса в заголовке журнала и при наличии строки **ctx** — `[x]`).

### Одна фраза: старт, промежуток или финал

Если пользователь пишет **только** «зафиксиру�� сессию» / «фиксация сессии» без уточнения:

1. **Явные сигналы к MID** — «промежуточно», «checkpoint», «пока не закрываем», «сохрани как есть продолжим позже» (без закрытия чата как финала).
2. **Явные сигналы к START** — «только старт», «начали», «отметь открытие», «пока без итога»; или в треде нет итога и пользователь отвечает на уточнение «только старт в дне и карточке?» — да.
3. **Явные сигналы к CLOSE** — перечень сделанного, артефакты, «и��ог», «закрываем»; или в файле дня уже есть **открытый** блок по `[[Objective]]` в «Заметках» (🔵/🟡) / опционально `- [ ]` в **ctx** — и пользователь приносит **итог** — по умолчанию **CLOSE** (не плодить второй открытый блок без дискриминатора).
4. **Неоднозначно** — **один** вопрос: «Фиксируем **только старт** (карточка + журнал в дне), **промежуточный** итог (ПЗ/Notes + строки в журнале) или **финал** со сменой статуса в заголовке журнала?»

**Запрет:** не считать выполненной фиксацией сессии запись **только** в `session-handoff` без правок **Objective** и **`YYYY/01-Daily/YYYY-MM-DD.md`** (кроме явной просьбы пользователя только на текст моста).
Журнал сессий — в **`## Заметки в течение дня`**; `session-handoff` — только чат-мост после vault.
**Порядок:** сначала vault (ПЗ, Objective, день), затем по умолчанию **CLOSE-5** (чат-мост); см. **CLOSE-6**.

## Режим диалога

- **Один шаг за ответ** там, где нужен выбор пользователя; ответ заканчивается вопросом или явным приглашением.
- **Правки в vault** (штамп в Objective, ПЗ, Notes) — **только после подтверждения** пользователем черновика (дата, путь к Objective, текст строк), либо по явному «да, запиши» / «зафиксируй ст��рт».
- **Голос / сырая транскрипция:** считать тем же вводом, что и текст: слегка нормализовать формулировку (без искажения смысла), **не** приписывать `ProjectId` или Objective при сомнении — **переспросить** одним коротким вопросом; в vault **не** писать до подтверждения черновика.

## Журнал се��сий в дне: `## Заметки в течение дн��`

**Назначение:** одним взглядом видеть, **над какой задачей** шла работа, **в каком состоянии** сессия и **когда** был последний заход.

### Карта статусов (circle-эмодзи в скобках заголовка `### HH:mm … (…)`)

| Эмодзи | Статус | Когда ставить |
|--------|--------|----------------|
| 🔵 | **Старт** | Первый заход по этой **Objective** в **этот календарный день** |
| 🟡 | **Продолжение** | Повторны�� заход по **той же** Objective в тот же день (новый заголовок `###` с **новым** `HH:mm`) |
| 🟢 | **Выполнена Objective** | Пользователь подтвердил закрытие / выполнение **Objective** |
| 🟣 | **DoD на сегодня завершен** | На сегодня закрыт **DoD**, сама Objective может оставаться открытой |
| ⚪ | **Отложена (на будущее)** | Осознанная пауза, перенос фокуса |
| 🔴 | **Не сработало** | Попытка не дала результата; в **Notes/ПЗ** — что пробовали и смена курса |

### Резюме сессии: «продолжаем» vs «работаем с другой задаче��»

- **Продолжаем ту же Objective** (тот же день, новый заход в чате): новый заголовок `### HH:mm [[та же Objective]] (🟡 Продолжение)` **или** (если открыт блок 🔵/🟡 без финала) дописать под ним промежуточные строки по **MID-6** без нового `###`.
- **Переключаемся на другую Objective**: новый заголовок `### HH:mm [[другая Objective]] (🔵 Старт)` — отдельный блок; предыдущую при **CLOSE** закрыть смено�� статуса в скобках (**CLOSE-7**).

## Подпротокол MID (промежуточная фиксация)

Сессия **не** считается закрытой: чекбокс в дне **остаётся** `- [ ]`. Итог позже — **CLOSE**.

### MID-1. Привязка

Как **CLOSE-1**: Objective, `ProjectId`, ПЗ при необходимости.

### MID-2. Проект и ПЗ

Как **CLOSE-2**. Если ПЗ нет и пользователь не хочет создавать — только **MID-3** (Notes).

### MID-3. Черновик ПЗ

В **`## Ход работы`** одна или несколько строк с явной пометкой **`(промежуточно, YYYY-MM-DD HH:mm)`** — что сделано; среда Cursor/CC/офлайн; артефакты — **wikilinks** `[[…]]`. При значимых решениях можно кратко дублировать смысл в **`## Ключевые решения`** с той же пометкой.

### MID-4. Черновик Objective

В **`## Notes`** дописать подзаголовок **`### Промежуточно (YYYY-MM-DD HH:mm)`** или маркированный пункт с тем же ярлыком — краткое резюме; только **wikilinks** на артефакты.

### MID-5. Вопрос о следующих шагах (опционально)

Не обязателен. По желанию одно короткое предложение: «Записать 1–3 следующих шага в Notes/ПЗ? (или „нет")». При «нет» — не добавлять пустой блок.

### MID-6. Ежедневная заметка: промежуточно в журнале (сессия открыта)

1. Дата и файл дня — как **START-1**, **DAY-RESOLVE**; при отсутствии файла — создать минимальный markdown.
2. В **`## Заметки в тече��ие дня`** найти **последний** блок по этой `[[Objective]]`, у которого в заголовке `###` статус **🔵 Старт** или **🟡 Продолжение**.
3. **Под** этим заголовком дописать **две** строки:

   ```markdown
   - [x] <краткое резюме промежутка>
   - [ ] Следующая задача: <кратко>
   ```

4. Если **открытого** блока нет — выполнить **START-7b** в режиме **🟡 Продолжение** и затем пункты 3.
5. **Опционально ctx:** если есть строка **START-7c** — оставить `- [ ]`.

### MID-7. Подтверждение и запись

Показать черновик (ПЗ, Notes, предлагаемые строки в журнале дня). После «да» / «запиши» — порядок: **ПЗ → Objective → файл дня** (только **MID-6** в `## Заметки в течение дня`). **Не** выполнять **CLOSE-7**. **CLOSE-8** — только если пользователь явно просит.

### MID-8. Файл session-handoff (MID)

По умолчанию при **MID** — **не** писать чат-мост. По **явной просьбе** пользователя — краткий блок по тому же шаблону, что **CLOSE-5**, в `.claude/session-handoff.md`.

## Подпротокол CLOSE (фиксация результата)

### CLOSE-1. Привязка

Определить Objective и `ProjectId` (как START-2): открытые файлы, сообщение пользователя, grep.

### CLOSE-2. Проект и ПЗ

Загрузить карточку проекта; найти кандидатов ПЗ в папке проекта. При >1 файле — выбор пользователя. ПЗ нет — предложить создать.

### CLOSE-3. Черновик ПЗ

Минимум строка в **`## Ход работы`** (дата, что сделано, среда Cursor/CC/офлайн). При значимых решениях — **`## Ключевые решения`**. Созданные/изменённые файлы — **`## Артефакты`** или список **wikilinks** `[[путь/к/файлу]]`; **не** backticks.

### CLOSE-4. Черновик Objective

Блок **`## Notes`** (или согласованный раздел): резюме сессии, ссылки на артефакты — **только wikilinks** `[[…]]`.

### CLOSE-4a. Следующие шаги — обязательный явный вопрос

**Всегда** перед финальным подтверждением записи (CLOSE-6) нужно зафиксировать намерение по продолжению работы:

1. После предъявления черновиков CLOSE-3 и CLOSE-4 **отдельным в��просом** спросить:
   **«Какие следующие шаги (1–3) записать в `## Notes` этой Objective и при необходимости в ПЗ? Если не фиксируем — ответьте «нет» или «пропуск».»**

### CLOSE-5. Чат-мост: session-handoff (по умолчанию)
