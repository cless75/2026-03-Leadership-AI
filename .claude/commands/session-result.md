# Команда: Фиксация результата сессии (session-link CLOSE)

**Навык:** [session-link](.cursor/skills/session-link/SKILL.md)

**Важно:** резюме **только** в `session-handoff` **не заменяет** фиксацию в vault. Основной исход — **ПЗ**, **Objective**, **`YYYY/01-Daily/YYYY-MM-DD.md`**: **CLOSE-7** — смена статуса в заголовке журнала в **`## Заметки в течение дня`** (🟢 / 🟣 / ⚪ / 🔴) и при ведении **ctx** — `[x]`. После vault по умолчанию — **CLOSE-5** (краткий чат-мост в `.claude/session-handoff.md`), кроме явного opt-out.

**Жёсткое правило:** подробный журнал сессии — в **`## Заметки в течение дня`**; `ctx` — опциональный индикатор.

Vault: **504-demo**. Ежедневные заметки: **`YYYY/01-Daily/YYYY-MM-DD.md`**.

## Входные данные ($ARGUMENTS)

- Текст итога, пути к изменённым файлам, `ProjectId`, ссылка на Objective.
- Синонимы: «зафиксируй результат», «фиксируем работу», «handoff» в контексте задачи.

## Обязательные источники

1. `.cursor/skills/session-link/SKILL.md` — подпротокол CLOSE.
2. `.cursor/skills/session-link/reference.md` — журнал, ctx, ПЗ.

## Режим

- Один шаг за ответ при уточнениях; объединённый черновик ПЗ + Objective — затем **обязательный явный вопрос о следующих шагах** (CLOSE-4a): что записать в Notes/ПЗ (1–3 пункта или «нет»).
- Порядок записи: ПЗ → Objective → файл дня (CLOSE-7) → session-handoff (CLOSE-5).

## Итог

Резюме: что записано, пути к ПЗ и Objective.

- **CLOSE-7**: обновить заголовок сессии в **`## Заметки в течение дня`** (🔵/🟡 → 🟢/🟣/⚪/🔴).
- **CLOSE-8**: при внеплановых действиях на день — **`### Внеплановые задачи`** с wiki-ссылками `[[…]]`.
