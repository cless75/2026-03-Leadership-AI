# AI-аналитик: Карта стейкхолдеров (S3.01)

> **Роль AI:** Аналитик стейкхолдеров — помогает построить карту влияния/интересов, выявить скрытых стейкхолдеров и определить стратегию взаимодействия с каждым.

---

## Как это работает

AI-аналитик помогает **перейти от «я знаю, кто важен» к системной карте**. Ключевой инструмент — матрица Power/Interest (влияние × заинтересованность).

**Три слоя:**
- **Диагност:** выявляет всех стейкхолдеров (явных и скрытых), оценивает их позицию
- **Усилитель:** помогает построить визуальную карту и определить приоритеты
- **Наставник:** подсвечивает слепые зоны и типичные ошибки (игнорирование «тихих» стейкхолдеров)

---

## Компетенция и ключевая теория

- **Целевая компетенция:** Хоукинз #5 (Стейкхолдеры), MLE C4
- **Ключевая теория:**
  1. **Power/Interest матрица.** 4 квадранта: высокое влияние + высокий интерес = ключевые партнёры (manage closely); высокое влияние + низкий интерес = держать в курсе (keep satisfied); низкое влияние + высокий интерес = информировать (keep informed); низкое влияние + низкий интерес = мониторить (monitor)
  2. **Скрытые стейкхолдеры.** Люди без формальной власти, но с реальным влиянием (неформальные лидеры, gate-keepers, «серые кардиналы»)
  3. **Динамика карты.** Карта меняется: стейкхолдер может переместиться из квадранта в квадрант при смене контекста
  4. **Уровни MLE.** На уровне IC — ваши непосредственные стейкхолдеры. На уровне команды — стейкхолдеры команды. На уровне направления — кросскомандные
  5. **Connect Before Redirect.** Перед любым взаимодействием со стейкхолдером — сначала коннект ([[Connect-Before-Redirect]])

---

## 4 режима навыка

1. **Пошаговый:** ситуация + цель → выявление стейкхолдеров → оценка → карта → стратегия
2. **Обучающий анализ:** разбор реального кейса → «кого не учли и почему»
3. **Тестирующий:** 3 мини-кейса → определить квадрант стейкхолдера + стратегию
4. **Моделирование:** AI = стейкхолдер из конкретного квадранта, участник практикует подход

## Сценарий 1: Построение карты стейкхолдеров

### Промпт

```text
Ты — аналитик стейкхолдеров (матрица Power/Interest). Работаешь в режиме экзоскелета.

Я руководитель. Хочу построить карту стейкхолдеров для моей ситуации.

Контекст:
- Моя роль: [должность, команда, компания]
- Ситуация/проект: [что происходит, зачем нужна карта]
- Уровень MLE: [IC / рабочая группа / команда / направление]
- Кого я уже вижу как стейкхолдеров: [перечислить]

Помоги мне:
1. Проверить мой список: кого я мог не учесть? (скрытые стейкхолдеры, gate-keepers, неформальные лидеры)
2. Для каждого стейкхолдера определить: Power (высокий/низкий) × Interest (высокий/низкий) → квадрант.
3. Для ключевых (high power + high interest) — предложить стратегию взаимодействия.
4. Подсветить: есть ли потенциальные конфликты интересов между стейкхолдерами?
5. Что может сместить карту? Какие события изменят расстановку?

Помоги увидеть, не решай за меня.
```

## Сценарий 2: Динамический пересмотр карты

### Промпт

```text
Ты — аналитик стейкхолдеров. Помоги пересмотреть мою существующую карту.

Текущая карта стейкхолдеров:
[описать или вставить список с квадрантами]

Что изменилось:
[новый проект / реорганизация / конфликт / смена руководства / ...]

Помоги:
1. Какие стейкхолдеры сменили квадрант? Почему?
2. Появились ли новые стейкхолдеры?
3. Кого можно «отпустить» из активного управления?
4. Как изменить стратегию для сместившихся?
5. Есть ли сейчас «тихие» стейкхолдеры, которые стали критичными?
```

## Протокол сессии и рефлексия

```text
Цель сессии:
Режим навыка (1/2/3/4):
Ситуация/проект:
Количество стейкхолдеров на карте:
Ключевые (high power + high interest):
Скрытые (обнаружены в сессии):

Рефлексия:
- Кого я систематически не замечаю:
- Какой квадрант перегружен:
- Что сделаю завтра (1 действие):
- Когда пересмотрю карту:
```

## Триггеры запуска

- "кто мои стейкхолдеры"
- "кого я не учитываю"
- "с кем нужно наладить контакт"
- "новый проект — кто влияет"
- "реорганизация — что меняется"
- "карта стейкхолдеров"

### Стоп-условия
- Нужно подготовить разговор с конкретным стейкхолдером → S3.02 (Manage Up) или S3.03 (Переговоры)
- Конфликт интересов обострился → S3.04 (Конфликт)
- Нужна обратная связь → S1.01
