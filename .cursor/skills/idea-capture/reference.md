# Справочник: idea-capture

**Навык:** [SKILL.md](SKILL.md)
**Команда:** [.claude/commands/idea-capture.md](.claude/commands/idea-capture.md)

## Триггеры

- Начало сообщения с **«Идея»** + разделитель (`:`, `—`, `–`, `-`) или пробел + текст
- Фразы: «зафиксируй идею», «сохрани мысль», «запиши мысль», «идея:», «идея по модулю/навыку X»

## Куда записывается

| Сценарий | Где | Что |
|---|---|---|
| 1 — короткая | `YYYY/01-Daily/YYYY-MM-DD.md` → `## Входящие дня` | Одна строка `- [ ] {текст} → Chat ⏰ HH:MM` |
| 2 — развёрнутая | `My-Notes/{YYYY-MM-DD}-Idea-{slug}.md` + ссылка в daily | Отдельный файл с frontmatter и заголовком |
| 3 — про модуль | daily со ссылкой на `[[504-M{N}-AI-Skill-...]]` или файл `My-Notes/M{N}-Ideas.md` | В зависимости от объёма |

## Формат строки в daily

**Базовая:**

```markdown
- [ ] {краткий текст} → Chat ⏰ HH:MM
```

**С источником-навыком:**

```markdown
- [ ] {текст} → [[504-M3-AI-Skill-Stakeholder-Mapping]] ⏰ HH:MM
```

**С ценностью / «Зачем»** (опционально):

```markdown
- [ ] {текст идеи} → Chat ⏰ HH:MM
  - *Зачем:* {короткий ответ}
```

**Ссылка на отдельную заметку** (сценарий 2):

```markdown
- [ ] [[2026-04-25-Idea-stakeholder-sync|Синхронизация стейкхолдеров перед M4]] → Chat ⏰ HH:MM
```

## Frontmatter отдельной заметки (сценарий 2)

```yaml
---
tags: [idea, 504]
CDate: YYYY-MM-DD
Day: "[[YYYY-MM-DD]]"
---
```

## Создание секции `## Входящие дня`

Если в файле дня секции нет — создать **перед** `## Заметки в течение дня` (если та есть) или в конце файла. Минимально:

```markdown
## Входящие дня

- [ ] {текст идеи} → Chat ⏰ HH:MM
```

## Альтернативы в Obsidian (если используются)

Если у участника настроен core-plugin «Daily notes» — сегодняшний файл создаётся автоматически по шаблону. Навык idea-capture пишет в тот же файл после его открытия.

## Что не поддерживается в этом пакете

- Общий Inbox проекта (`# 📥 Inbox` в карточке проекта) — карточек проектов в пакете нет.
- Единый лог идей (`02-CoreAutomation/186/Idea-Capture-Log.md`) — файла нет, таких лог-файлов не создаём.
- Subagent через `mcp_task` — инструмент недоступен в Claude Code; правки в skill-файлах делает агент напрямую после подтверждения.
- Сократовский диалог с предположением про «направление и проект» — у пакета нет понятия «направление». По запросу участника можно одним вопросом уточнить «ценность реализации» — не разворачивать в длинный диалог.
