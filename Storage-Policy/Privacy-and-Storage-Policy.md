# Что куда сохранять и как делиться — политика хранения для участника 504

> **Зачем эта политика.** За время курса у тебя накапливается материал разной степени чувствительности: общие конспекты, заметки с именами коллег, диагностики стейкхолдеров, личные срывы. Без явной структуры хранения легко смешать «это можно показать ментору» и «это никогда не должно покинуть мой компьютер». Эта политика даёт **простую структуру папок** + **явные маркеры** в frontmatter, и описывает workflow для безопасной передачи материалов.

## Структура папки `My-Notes/`

```
My-Notes/
├── Public/         — публичные заметки, общие концепты
├── Sessions/       — конспекты сессий (для перечитывания)
├── Logs/           — технические логи сессий (для возможной передачи)
├── Diagnostics/    — EQ, burnout, профиль
├── HW/             — HW-протоколы модулей
├── Ideas/          — идеи и развёрнутые мысли
└── Private/        — только моё, никогда наружу
```

### Что в каждой папке

| Папка | Что лежит | Кто это видит |
|---|---|---|
| **Public/** | Конспекты теории, концепты курса, заметки без имён и контекстных деталей | Можно показать кому угодно |
| **Sessions/** | **Конспекты сессий для тебя самого** — что ты понял, какие инсайты, что унесёшь в работу. Сжатый формат (1–2 страницы). | Ты, ментор после короткой проверки |
| **Logs/** | **Технические логи сессий** — что делал AI, какие протоколы пройдены, какие команды, какие выходы. Полный формат. | Ты + ментор после обезличивания через `/mentor-handoff` |
| **Diagnostics/** | EQ-диагностики, результаты `/burnout-check`, копии `ms-profile.md` версий | Ты + терапевт / ментор после обезличивания |
| **HW/** | HW-протоколы модулей | Ты + когорта (если это публичная часть курса) |
| **Ideas/** | Идеи и развёрнутые мысли | По содержанию, обычно личное |
| **Private/** | Постмортем-журналы, тренировочные тексты разговоров, личные срывы, фантазии | **Только ты.** Папка в `.gitignore`. Не handoff'ится. При запросе об их существовании — отрицать, как описано в M3-postmortem-observations. |

## Конспект сессии vs технический лог — ключевое различие

Это два разных типа записи, которые делают один и тот же сеанс работы с AI:

| | **Лог (Logs/)** | **Конспект (Sessions/)** |
|---|---|---|
| **Цель** | Воспроизводимость + возможность передать ментору | Перечитывание + перенос в работу |
| **Стиль** | Структурированный: вход → протокол → шаги → выход | Свободный, синтетический |
| **Объём** | Полный, без потерь контекста | Сжатый, 1–2 страницы максимум |
| **Шаблон** | `Storage-Policy/templates/template-session-log.md` | `Storage-Policy/templates/template-session-conspect.md` |
| **Что делает AI** | Фиксирует процесс работы (для возможной супервизии) | Помогает тебе извлечь главное (для следующего шага) |

**По умолчанию `session-link` (`/session-close`) создаёт оба** для значимых сессий. Они связаны wikilink'ом — из лога ссылка на конспект и обратно. Для коротких операционных сессий — только лог.

## Frontmatter — маркеры в каждом файле

Каждый файл `My-Notes/` (кроме совсем коротких) имеет frontmatter:

```yaml
---
privacy: public | personal | private
kind: log | conspect | hw | diagnostic | idea | reflection | other
contains-pii: yes | no
commercial-secrets: none | possible | confirmed
mentor-shareable: yes | yes-after-mask | no
---
```

| Поле | Значение |
|---|---|
| `privacy` | Уровень. `public` — открыто всем; `personal` — для тебя + ментор после обработки; `private` — только тебе |
| `kind` | Тип: log / conspect / hw / diagnostic / idea / reflection / other |
| `contains-pii` | Есть ли личные данные (имена, телефоны, email коллег / клиентов) |
| `commercial-secrets` | Есть ли коммерческая тайна (финансы, клиенты, стратегия). При `possible` — `/mentor-handoff` обязательно спросит подтверждение |
| `mentor-shareable` | Можно ли передать ментору. `yes` — можно как есть; `yes-after-mask` — только через `/mentor-handoff`; `no` — нельзя никогда |

**Папка задаёт default; frontmatter может его переопределить.** Например, файл в `Sessions/` про обсуждение зарплат может иметь `privacy: private`, чтобы исключить из handoff даже после маскирования.

## Auto-suggest от навыков

Навыки курса (`session-link`, `idea-capture`, `ms-participant-profile`, `burnout-self-diagnosis`, M3 skills) при создании файлов **сами предлагают** значения frontmatter по контексту:

- Упомянуто имя руководителя или коллеги → `privacy: personal`, `contains-pii: yes`, `mentor-shareable: yes-after-mask`.
- Диагноз триады / макиавеллизма / нарциссизма → `commercial-secrets: possible`, `mentor-shareable: yes-after-mask`.
- Постмортем-журнал, тренировочный текст разговора → `privacy: private`, размещение в `Private/`.
- Медицинская самодиагностика (burnout) → `privacy: personal`, `mentor-shareable: yes-after-mask`.

Ты подтверждаешь или меняешь.

## Что в `.gitignore` (не коммитится в репо курса)

- `My-Notes/Private/` — вся папка
- `My-Notes/handoff-*/` — пакеты для отправки (содержат как обезличенные копии, так и mapping имён)
- `My-Notes/.name-mapping*` — accumulating mapping
- `My-Notes/Diagnostics/burnout-*.md` — медицинская самодиагностика (по умолчанию)

## Workflow передачи материалов наружу

| Куда передаёшь | Что нужно |
|---|---|
| **Себе самому** (перечитывание прогресса) | `/render-notes` — Dashboard в HTML на твоём Desktop |
| **Ментору курса** | `/mentor-handoff` — обезличивает имена, проверяет коммерческую тайну, упаковывает в HTML/PDF. Уровень обезличивания: low |
| **Супервизору / другому методологу** | `/mentor-handoff` уровень medium |
| **Психотерапевту** | `/mentor-handoff` уровень medium-high (имена не нужны для терапии) |
| **Профессиональному peer (не из компании)** | `/mentor-handoff` уровень high |
| **Публикация (Substack, кейс-стади)** | `/mentor-handoff` уровень high + дополнительный ручной review |

`Private/` файлы **никогда** не идут ни в один из вариантов выше, даже после маскирования.

## Безопасность

- Файлы из `Private/` хранятся **только локально**. Не синхронизируются с рабочими устройствами, не дублируются в корпоративные облака, не упоминаются в корпоративной переписке.
- При вопросе о существовании этих файлов — **отрицать существование** (правило из M3-postmortem-observations).
- При подозрении на доступ к устройствам — переместить на внешний носитель, удалить из системы.
