# Команда: Старт сессии по Objective (session-link START)

**Навык:** [session-link](.cursor/skills/session-link/SKILL.md)

Инициализация работы над задачей: выбор Objective (при необходимости), открытие карточки в **Cursor** / контекст в **CC**, открытие в **Obsidian** (если доступен CLI), создание файла дня при отсутствии, **штамп старта сессии** со ссылкой на `[[YYYY-MM-DD]]` в карточке (после подтверждения), **журнал** в **`## Заметки в течение дня`** (**START-7b**: заголовок `### HH:mm [[Objective]] (🔵 Старт|🟡 Продолжение)` + строка «Начало работы»), краткий контекст ПЗ/проекта.

**Жёсткое правило:** канон фиксации в дне — **`## Заметки в течение дня`** в `YYYY/01-Daily/YYYY-MM-DD.md`. Секция **ctx** — **опциональный** индикатор.

Vault: **504-demo**. Ежедневные заметки: **`YYYY/01-Daily/YYYY-MM-DD.md`**.

## Входные данные ($ARGUMENTS)

- Необязательно: `ProjectId` / номер проекта, путь к Objective, формулировка «работаем над …», дата (`YYYY-MM-DD` или «сегодня»).
- Без аргументов — определить сценарий по первому ответу пользователя.

После выбора Objective — вопрос о **ценности сессии** (**START-2b**): один короткий ответ или «пропуск».

## Обязательные источники

1. `.cursor/skills/session-link/SKILL.md` — подпротокол START.
2. `.cursor/skills/session-link/reference.md` — шаблоны журнала, статусы.

## Режим

- Один шаг за ответ; не записывать в vault без подтверждения черновика.
- После подтверждения: правка **Objective** (секция `### Сессии`) **и** **`YYYY/01-Daily/YYYY-MM-DD.md`** — блок **START-7b**; при необходимости **START-7c** в **ctx**; создание файла дня при отсутствии.

## Итог

Кратко: какая Objective, день, открыто ли в IDE/Obsidian, записан ли штамп в карточку и блок журнала.
