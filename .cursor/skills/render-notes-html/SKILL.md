# Навык render-notes-html (Dashboard прогресса)

**Справочник (структура данных, источники, fallback):** [reference.md](reference.md)
**Скрипт:** `templates/render-script.py` — Python-генератор; запускается через wrapper-команду `/render-notes`.

## Назначение

Дать участнику быстрый и красивый способ перечитать свой прогресс и историю взаимодействия с AI **за период**. Не для передачи кому-либо — только для себя. Если нужна передача — `mentor-handoff`.

## Источник политики приватности

`Storage-Policy/Privacy-and-Storage-Policy.md` курса. Навык использует папочную структуру `My-Notes/` + frontmatter privacy, чтобы корректно проставить бейджи и баннеры.

## Состав Dashboard

| Секция | Источник данных |
|---|---|
| **Шапка** | Псевдоним (из `ms-profile.md` или `my-profile.md`), период (день/неделя/модуль), дата генерации. Баннер «**содержит Private — не передавать**» если в выборку попали Private-файлы |
| **Текущий статус** | Burnout-зона из последнего `My-Notes/Diagnostics/burnout-*.md`, текущая версия профиля, активная программа MS, флаги `health_signal` |
| **Прогресс EQ** | Если ≥2 EQ-диагностик — мини-таблица «компонент: до → после» + стрелки роста/падения. Inline SVG-график для ≤5 точек |
| **Timeline** | Все Logs + Sessions + Diagnostics + HW + Ideas из `My-Notes/` за период, отсортированы по дате. Бейджи privacy и kind. Расхлоп-секции содержимого (`<details>`) |
| **Активные эксперименты** | Из секций «Эксперименты» в M3-S0.03 / shadow / профиля — список открытых с датами начала и due-датами |
| **Открытые вопросы** | Из `open-questions.md` если есть — список со статусами |
| **Decisions log** | Если есть `decisions-log.md` — последние 5–10 решений |
| **Footer** | Статистика: сколько сессий (логов / конспектов), сколько HW, сколько идей, средний интервал между сессиями |

## Режимы

1. **`/render-notes day`** — за конкретный день (по умолчанию — сегодня). Удобно для вечернего обзора.
2. **`/render-notes week`** — последние 7 дней или текущая ISO-неделя. **По умолчанию при вызове без аргумента.**
3. **`/render-notes module M2`** — все материалы по модулю (M1/M2/M3/M4).
4. **`/render-notes range YYYY-MM-DD..YYYY-MM-DD`** — произвольный диапазон.

## Размещение результата

`Desktop/<participant>/dashboard-YYYY-MM-DD.html` (НЕ в репо курса).

- `participant` берётся из псевдонима `ms-profile.md` или `my-profile.md`.
- Если имя не известно — спросить на первом запуске и сохранить в `.claude/settings.local.json` ключом `PARTICIPANT_DESKTOP_FOLDER`.
- HTML самодостаточный (inline CSS); открывается двойным кликом в любом браузере.
- Папка с рендерами в `.gitignore` (`**/.html-renders/`) — на случай если кто-то держит копии в репо.

## Бейджи приватности и баннер

На каждой timeline-секции — цветной бейдж по `privacy` frontmatter / папке:

- 🟢 `public` — public
- 🟡 `personal` — personal
- 🔴 `private` — private

Если в выборку попал ≥1 файл с `privacy: private` или из `My-Notes/Private/` — **в шапке HTML баннер**:

> **⚠️ Этот рендер содержит Private-файлы. Не передавать никому. Перечитывать только локально.**

Это намеренно сильный сигнал — защита от случайной отправки кому-либо.

## Шаги выполнения навыка

1. **Период.** Распознать аргумент (`day` / `week` / `module N` / `range A..B`), при отсутствии — `week`.
2. **Псевдоним.** Прочитать из профиля или спросить (один раз на участника).
3. **Сбор файлов.** Glob по `My-Notes/Public/`, `Sessions/`, `Logs/`, `Diagnostics/`, `HW/`, `Ideas/`. Опционально `Private/` — только если участник явно попросил «include private» (по умолчанию исключаются).
4. **Парсинг frontmatter** каждого файла: privacy, kind, date.
5. **Сбор Dashboard-данных.** Для каждой секции — соответствующий запрос (см. таблицу выше + reference.md).
6. **Запуск Python-скрипта.** `templates/render-script.py` принимает входы (список файлов + dashboard-data + псевдоним + период) и собирает HTML на Desktop.
7. **Открытие в браузере** через `Invoke-Item` или эквивалент.
8. **Подтверждение участнику** — путь к файлу + информация (период, сколько секций, наличие Private-баннера).

## Что НЕ делает

- Не передаёт никуда наружу. Если нужно ментору — `mentor-handoff`.
- Не редактирует исходные файлы.
- Не агрегирует данные больше чем за 1 период за раз.
- Не ставит диагнозы по содержимому. Просто визуализирует.
