# Команда: eq-self-diagnosis

Проведи протокол **самодиагностики EQ** по навыку **eq-self-diagnosis** (модель Гоулмана: пять компонентов, слепые вопросы 1–5, три ловушки, паутинка, одна точка фокуса). Доступны **мини-кейсы 1–4** из `eq-mini-cases.md` и сценарии в `eq-competency-scenarios.md`; режимы **EQ-C** (сценарий по компоненту) и **EQ-M** (наставник); **после** раскрытия маппинга и паутинки — справочный трек из `eq-concepts-reference.md` (концепты, эмоции). По запросу или после фокуса — **T1**, при неудаче — **T2**. Опционально — три вопроса Soft Skills / Хоукинз (EQ-6).

## Входные данные

Контекст: $ARGUMENTS

Если не указано — запроси:

1. **Режим:** своя ситуация (A) **или** мини-протокол с кейсом (B: мини-кейсы 1–4) **или** только **T2** / **T1** **или** **EQ-C** / **EQ-M** (указать компонент).
2. Для режима A — кратко: что случилось, роль, что уже сделано.

## Методология

**Канон навыка:** [.cursor/skills/eq-self-diagnosis/SKILL.md](.cursor/skills/eq-self-diagnosis/SKILL.md)

**Приложения:** `eq-mini-cases.md`, `eq-competency-scenarios.md`, `eq-concepts-reference.md` в той же папке.

## Поведение

- Слепая фаза EQ-2: не называть компоненты до шага EQ-4; вопросы 1–5 — как в SKILL.
- После EQ-5 **предложить** справочно-обучающий трек (EQ-7): концепты, лексика эмоций, углубление по фокусу.
- Секция «Контакты» в конце SKILL — выводить по завершении сессии или по запросу.
- Не ссылаться на другие навыки Cursor из этого протокола.
