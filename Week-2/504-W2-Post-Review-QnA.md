## Блок 1: Диагностика команды и Такман (S2.01)

### Кейс: «Группа = конвейер, команда = пит-стоп с невесомостью» (Алексей, W2)

> Алексей описал разницу: рабочая группа работает как конвейер (каждый делает своё), а команда — как пит-стоп (координация в невесомости, каждый знает роль).

**Промпт:**

```text
Ты — аналитик командной динамики (S2.01, модель Такмана).

Метафора из W2: «группа = конвейер (каждый делает своё, последовательно), команда = пит-стоп с невесомостью (синхронная координация, каждый знает роль)».

У меня есть [команда / рабочая группа]. Помоги разобраться:

Контекст:
- Тип: [команда / рабочая группа / артель]
- Размер: [N человек]
- Как давно вместе: [срок]
- Что наблюдаю: [поведение]

Помоги:
1. Это группа или команда? По каким признакам?
2. На какой фазе Такмана мы (Forming/Storming/Norming/Performing)?
3. Если группа — нужна ли она команда? Или группа эффективнее для этих задач?
4. Что конкретно изменить, чтобы перейти к следующей фазе?
```

## Блок 2: Value Chain и процессы (S2.00)

> Денис работает в e-commerce. Хочет построить карту цепочки ценности для своего направления.

**Промпт:**

```text
Ты — аналитик цепочки ценности (S2.00, Value Chain Diagnostics).

Я работаю в e-commerce. Хочу построить карту цепочки ценности для моего подразделения.

Контекст:
- Моя роль: [должность]
- Направление: [что делаем — маркетинг / логистика / продукт / ...]
- Клиент цепочки: [кто получает результат нашей работы]
- Основные шаги (как я их вижу): [перечислить]

Проведи диагностику по 6 шагам S2.00:
1. Границы цепочки (вход → выход)
2. Шаги (кто, что, кому)

4. Узкие места (bottleneck)
5. Точка расширения (где маленькое изменение → большой эффект)
6. Эксперимент (что изменить на этой неделе)
```

## Блок 4: Организационные формы (из W2 Workshop — незапланированная тема)

### Кейс: «Три формы: артель, рабочая группа, команда» (Дмитрий, W2)

> На W2 Дмитрий ввёл дифференциацию: артель (каждый сам, общий ресурс), рабочая группа (координация менеджером), команда (общая цель, лидер, синергия).

**Промпт:**

```text
Ты — аналитик организационных форм (MLE Framework).

Три формы совместной работы:
- **Артель:** каждый работает сам, общий ресурс/пространство, нет лидера
- **Рабочая группа:** координация менеджером, задачи распределены, Run-режим
- **Команда:** общая цель, лидер, синергия, Grow-режим

Моя ситуация:
- Подразделение: [описание]
- Формально мы: [как называемся]
- По факту работаем как: [артель / группа / команда — и почему]

Помоги:
1. Диагностировать: какая форма сейчас по факту?
2. Какая форма НУЖНА для наших задач? (не всегда команда = лучше)
3. Если нужен переход (например, группа → команда) — что для этого нужно?
4. Какой уровень MLE здесь работает и какие навыки подключить?
```
