# Навык mentor-handoff (обезличивание и упаковка для передачи)

**Справочник (категории, шаблоны mapping, regex):** [reference.md](reference.md)
**Шаблоны:** `templates/handoff-checklist.template.md`, `templates/name-mapping.template.md`, `templates/commercial-secrets-review.template.md`

## Назначение

Превратить набор файлов участника (логи сессий, диагностики, HW-протоколы) в **обезличенный пакет**, который безопасно передать наружу — ментору, супервизору, терапевту, профессиональному peer или для публикации. Гарантировать:

- ни одно имя коллеги / руководителя / клиента не уйдёт без явной замены или подтверждения участника;
- потенциальная коммерческая тайна выявлена и обработана до отправки;
- файлы с пометкой `privacy: private` или из папки `My-Notes/Private/` **никогда не попадают** в выход.

## Источник политики приватности

Курс 504: `Storage-Policy/Privacy-and-Storage-Policy.md` в корне репо. Этот навык применяет политику; не подменяет её.

## 6-шаговый протокол

### Шаг 1. Вход

Получить от участника:

- **Список файлов или папка** для обработки. Принимаются абсолютные/относительные пути, маски (`My-Notes/Logs/M3-*.md`), либо отдельные файлы.
- **Что включать по умолчанию:** `My-Notes/Public/`, `My-Notes/Sessions/`, `My-Notes/Logs/`, `My-Notes/Diagnostics/` (по выбору), `My-Notes/HW/`. Никогда — `My-Notes/Private/`.

**Авто-проверка отказа:**

- Файл из `My-Notes/Private/` или с `privacy: private` в frontmatter → **исключить**, явно сообщить пользователю: «файл X помечен как только-личный, исключаю из handoff».
- Файл с `mentor-shareable: no` → **исключить**.
- Если в результате после исключений список пуст — пояснить и завершить.

### Шаг 2. Цель и уровень обезличивания

Запросить у участника одним вопросом с 5 опциями:

1. **Ментор курса** (низкий уровень обезличивания). Ментор уже в контексте; роли и обезличенные имена.
2. **Супервизор / другой методолог** (средний). Полное маскирование имён + типы компаний; коммерческая тайна — review.
3. **Психотерапевт** (средний-высокий). Имена не нужны для терапии; роли + ситуации; коммерческая тайна — удаляется.
4. **Профессиональный peer (вне компании)** (высокий). Имена + типы компаний обезличены; коммерческая тайна — обязательное удаление; финансы — обобщить.
5. **Публикация (Substack, кейс-стади, презентация)** (полный). Всё обезличено + ручной review автором; цитаты переформулированы; узнаваемые детали (отрасль, размер) обобщены.

Уровень управляет жёсткостью пунктов 3–4.

### Шаг 3. Auto-сканирование

По каждому файлу — поиск:

- **Заглавные имена собственные** (русские и латинские; ≥2 заглавных буквы в значимом контексте).
- **Названия компаний / продуктов** (по словарю + капитализация).
- **Финансовые числа** (≥6 цифр, проценты, валюты, оценки масштаба).
- **Email, телефоны, URL** (regex).
- **Триггерные фразы коммерческой тайны** (см. список в reference.md):
  - planning M&A, sale, IPO, привлечение инвестиций;
  - конкретные показатели выручки / маржи / зарплат / costs;
  - имена клиентов + договоров + цен;
  - стратегические решения (выход на рынок, закрытие направления);
  - кадровые решения (увольнения, зарплаты, продвижения) с конкретными именами.
- **Цитаты в кавычках**, потенциально вредящие репутации упомянутого человека.

Сводка по каждому файлу:

```
{file}: имена X (3), компании Y (1), финансы Z (2), цитаты W (4), триггеры commercial: 2
```

### Шаг 4. Mapping и контр-вопросы

**Имена.** Один диалог на все имена:

> «Найдены имена: Стас, Светлана, Влад, Максим, Алияр.
> Предлагаемые замены (на основе словаря reference.md):
> - Стас → СРО (формальный руководитель)
> - Светлана → СОО
> - Влад → PO смежного продукта
> - Максим → бывший непосредственный руководитель
> - Алияр → линейный коллега
> Подтвердить / изменить / оставить какое-то имя как есть?»

**Компании / продукты:** аналогично, с предложением заменить на роль / тип («компания A», «B2B-вендор», «крупный клиент», «продукт X»).

**Триггерные фразы коммерческой тайны.** Для каждой — выбор:

> «Найдена потенциально чувствительная фраза: «… *<цитата>* …». Категория: <финансы/клиенты/стратегия/кадры>. Действие:
> (a) оставить как есть (моя ответственность),
> (b) переформулировать обобщённо (предложу вариант),
> (c) удалить из выхода.
> Что выбираешь?»

**При confirmed коммерческой тайне** (3+ триггеров в одном файле или явный confirmed во frontmatter):

> «В файле X есть {N} признаков коммерческой тайны. Прежде чем продолжить — оцени:
>  - публикуется ли это в годовом отчёте компании / в публичных коммуникациях?
>  - есть ли у тебя NDA, который ограничивает обсуждение?
>  - возможен ли репутационный или юридический ущерб, если конкретные формулировки уйдут наружу?
>  
>  Если хотя бы на один ответ «нет» / «не уверен» — переформулировать или удалить.»

Двойное подтверждение перед продолжением.

### Шаг 5. Создание пакета

После согласования mapping:

1. Создать папку `My-Notes/handoff-YYYY-MM-DD-{цель}/` (где {цель} — `mentor` / `supervisor` / `therapist` / `peer` / `publication`).
2. Для каждого файла создать копию с применёнными заменами в этой папке.
3. Сохранить mapping в `My-Notes/handoff-YYYY-MM-DD-{цель}/.name-mapping.md` (по шаблону `templates/name-mapping.template.md`).
4. **Diff-обзор**: показать пользователю по каждому файлу: сколько имён заменено, сколько фраз удалено/переформулировано, что осталось как есть. Запросить финальное «да» или «вернуть и доработать».
5. После «да» — пакет считается готовым.

**Опционально — accumulating mapping.** Если участник подтвердил при первом запуске опцию «использовать общий mapping для всех handoff'ов», то mapping также синхронизируется с `My-Notes/.name-mapping.md`. При следующих запусках навыка предложенные замены берутся из общего mapping для consistency.

### Шаг 6. Упаковка в HTML/PDF

По умолчанию — собрать обезличенные `.md` в один HTML с TOC + бейджем «Обезличено для: <цель>, <дата>», экспортировать в PDF через headless Edge.

**Размещение результата:**

- Если `participant-name` известен (из `ms-profile.md` или `my-profile.md`): `Desktop/<participant>/handoff-YYYY-MM-DD-{цель}/<имя>.pdf` (+ `.html`).
- Иначе: `Desktop/504-handoff-YYYY-MM-DD-{цель}/<имя>.pdf`.
- На первом запуске — спросить, использовать ли участникова имя в путях.

**Правило файлов на Desktop:** см. feedback-правило `feedback_case_file_location` (если применимо в среде агента) — папки кейсов на Desktop, не в Downloads.

## Жёсткие правила

1. **Файлы из `My-Notes/Private/` или с `privacy: private`** — никогда не в handoff даже после mask. Если участник принудительно настаивает — двойное подтверждение «вы уверены, что хотите передать `<имя файла>` (постмортем-журнал и т. п.)?» с напоминанием о правилах безопасности.
2. **PII** (телефоны, email, паспортные данные, адреса, медицинские) — **удаление**, не маскирование.
3. **Файлы с `commercial-secrets: confirmed`** — двойное подтверждение перед включением.
4. **Mapping не коммитится в git** (.gitignore: `My-Notes/.name-mapping*`, `My-Notes/handoff-*/`). Сообщать участнику при создании.
5. **Финальный diff-обзор обязателен.** Без явного «да» — пакет не финализируется.
6. **Уровень обезличивания (Шаг 2) — диктует жёсткость Шагов 3–4.** Для публикации — самое строгое.
7. **Никаких «я уверен, всё ок».** Каждое решение — за участника, явно.

## Триггеры запуска

- «подготовь handoff»
- «отправить ментору / супервизору / терапевту / peer»
- «mentor handoff», «supervisor handoff»
- «обезличить и упаковать»
- «сделать кейс для публикации»
- «передача материалов терапевту»

## Стоп-условия

- Все входные файлы исключены из handoff (Private или mentor-shareable: no) — пояснить, завершить.
- Участник не подтверждает Шаг 4 (mapping или коммерческая тайна) — приостановить, не финализировать.
- Острая ситуация (физическая угроза, кризис) — рекомендовать прямой звонок ментору / врачу / в HR, не handoff через документы.

## Связи

- `Storage-Policy/Privacy-and-Storage-Policy.md` (в каждом курсе MS, использующем этот навык) — источник политики.
- `session-link` — создаёт log/conspect файлы, которые потом попадают в handoff.
- `render-notes-html` — параллельный навык для **самого участника** (Dashboard прогресса), не для передачи.

## Запреты

- Не передавать материалы без явного согласия участника на Шаге 4.
- Не работать без Storage Policy в курсе — если её нет, попросить участника зафиксировать минимальную (отметить какие файлы public/personal/private).
- Не угадывать mapping имён без подтверждения; не проставлять commercial-secrets: confirmed автоматически.
