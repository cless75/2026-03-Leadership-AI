# Source — S09: Диагностика цепочки создания ценности (M2)

> Сжатая выжимка навыка-оркестратора для NotebookLM. Полная версия с промптом для Claude Project: документ участника «S09 Диагностика цепочки создания ценности (Value Chain Diagnostics)» (GDoc из пакета M2).

## Зачем

Участник строит **карту цепочки создания ценности** (5–9 шагов от входа до результата), отмечает узкие места и точки, где AI может усилить процесс **без «коляски»** (экзоскелет: AI структурирует и подсвечивает — решение остаётся за человеком).

## Шесть шагов оркестратора

1. **Границы:** вход в цепочку, выход (ценность), кто получатель.
2. **Элементы:** 5–9 шагов; на каждом — артефакт, решение, кто вовлечён.
3. **Состояние шага:** кто делает, outcome, узкое место, тип задачи (стандарт / повтор / нестандарт).
4. **Три слоя AI на шаг:** Диагност (паттерны) / Усилитель (скорость, рутина) / Наставник (учит видеть системно); явно отметить, где AI = **коляска** (навык деградирует).
5. **Приоритизация:** impact × effort; стиль делегирования по Бланшару (S1–S4); можно ли собрать **AI-микрокоманду** (3 роли вокруг точки).
6. **Антипаттерны:** «золотая клетка» (застревание в Run), коляска, шаги без трения (AI не нужен).

## Связь с неделей M2

- **Хоукинз #3** — процессы и сотрудничество: цепочка показывает, *где* процесс буксует.
- Навыки **S05–S08** — точечные инструменты после карты: фаза Такмана, роли, ритмы, делегирование на конкретном узком месте.

## Ключевые формулировки для слайдов и аудио

- «Сначала карта цепочки — потом точечный эксперимент.»
- «Оркестратор задаёт вопросы, не строит цепочку за вас.»
- «Один value stream за сессию — не весь портфель.»
