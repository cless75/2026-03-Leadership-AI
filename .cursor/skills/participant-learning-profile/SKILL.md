# Навык: Учебный профиль участника (Learner Profile)

## Когда использовать

- После обработки транскрипции сессии (KickStart, воркшоп, лекция) — как финальный шаг
- При получении новых данных: чат участника, домашнее задание, рефлексия, self-assessment
- Периодически: ревью когорты перед следующей неделей (агрегация)
- Триггеры: «обнови профиль», «учебный профиль», «профиль участника», «learner profile», «cohort review»

**Оркестратор:** может вызываться из ms-process-lecture (навык не импортирован) или напрямую пользователем.

## Входы

| # | Вход | Обязательность | Формат |
|---|------|---------------|--------|
| I1 | **Идентификация участника** | Обязательно | Имя или ParticipantCode |
| I2 | **Транскрипт(ы) сессий** | Хотя бы один | Путь к `.md`/`.txt`/`.vtt` |
| I3 | **Чат-лог** | Опционально | Путь к `.md`/`.txt` (Zoom/TG export) |
| I4 | **Анализ сессии** | Опционально | Путь к `{ProjectId}-a-*-Analysis.md` |
| I5 | **Домашние задания / рефлексии** | Опционально | Путь к файлам или текст |
| I6 | **Self-assessment** (форма самоидентификации) | Опционально | Путь к `.md` или данные формы |
| I7 | **Существующий профиль** | Авто | Система ищет `{ProjectId}-profile-{Code}.md` |
| I8 | **ProjectId и контекст курса** | Обязательно | Число + путь к ПЗ или Cohort Map |

**Режим batch:** при запросе «обнови профили когорты» — применить ко всем участникам с имеющимися данными.

## Выход

### Индивидуальная карточка профиля

Файл: `2026/{ProjectId}/Profiles/{ProjectId}-profile-{ParticipantCode}.md`

Если папки `Profiles/` нет — создать.

**Структура карточки** (шаблон — см. `504-g-Learner-Profile-Standard.md`):

```yaml
---
tags: [learner-profile, {ProjectId}]
ProjectId: {ProjectId}
ParticipantCode: "{Code}"
EntrySegment: "MDPG | External | Alumni478 | Mixed"
CDate: {первое создание}
LastUpdated: {дата}
ProfileVersion: {N}
---
```

#### Обязательные секции карточки

1. **Идентификация** — код, сегмент, входной контекст (роль, компания, вызов)
2. **Компетенции Хоукинза (текущий уровень)** — таблица 5 компетенций × оценка по сигналам
3. **Паттерн участия (Engagement Pattern)** — стиль: голос/чат/молчание, роль в группе
4. **Вопросы и их уровень** — классификация заданных вопросов по глубине и теме
5. **Участие в кейсах** — роль, вклад, уровень применения моделей
6. **AI-зрелость** — текущий уровень и динамика
7. **Фокус развития** — приоритетные навыки (S01-S16), рекомендуемые AI-роли
8. **Риски и внимание** — сигналы выпадения, перегруза, стагнации
9. **Хронология эволюции** — запись по сессиям (KS → W1 → W2 → ...)

### Обновление Cohort Map

Файл: `2026/{ProjectId}/Deliverables/{ProjectId}-a-Cohort-Segment-Skill-Map.md`

Добавить/обновить строки в:
- Матрицу «сегмент × скилл» — если приоритеты изменились
- QualitySignals в логике профиля
- Секцию «История» — запись об обновлении

## Алгоритм

### Шаг 1: СБОР СИГНАЛОВ → Д1

1. Прочитать существующий профиль (I7), если есть
2. Прочитать входные данные (I2–I6)
3. Извлечь все реплики/сообщения участника из транскрипта и чата
4. **Д1 — Вывести карту данных:** какие источники доступны, за какой период, сколько реплик найдено. Ждать подтверждения

### Шаг 2: АНАЛИЗ ВОПРОСОВ

Для каждого вопроса участника классифицировать:

| Измерение | Шкала | Описание |
|-----------|-------|----------|
| **Глубина** | L1 / L2 / L3 | L1 = уточняющий (факт, процедура); L2 = концептуальный (почему, как связано); L3 = интегративный (вызов модели, синтез, перенос) |
| **Тема** | H1–H5, Meta | Маппинг на компетенции Хоукинза (#1 Миссия, #2 Коммуникация, #3 Процессы, #4 Обучение, #5 Стейкхолдеры) или Meta (о курсе/AI/инструментах) |
| **Тип запроса** | C / R / T / F | C = Clarification (уточнение); R = Reflection (рефлексия); T = Tool (запрос инструмента); F = Challenge (вызов, несогласие) |

**Формула сигнала:** больше L2–L3 и F-вопросов → участник на стадии осознанной компетентности или перехода. Много L1 и C → начальная стадия, нужен Наставник.

### Шаг 3: АНАЛИЗ УЧАСТИЯ В КЕЙСАХ

Для каждого кейса/упражнения/demo оценить:

| Измерение | Значения |
|-----------|----------|
| **Роль** | Инициатор / Со-ведущий / Активный участник / Наблюдатель / Отсутствовал |
| **Вклад** | Предоставил кейс / Дискутировал / Задал вопросы / Поддержал реакцией / Молчал |
| **Применение моделей** | Использовал модель из курса / Привёл аналогию / Не применял |
| **Качество** | Глубокий (аргументация, факты) / Средний (мнение, опыт) / Поверхностный (согласие, "+" в чате) |

### Шаг 4: КОМПЕТЕНЦИИ ХОУКИНЗА (текущий срез)

Для каждой из 5 компетенций Хоукинза определить уровень участника на основе собранных сигналов:

| Уровень | Описание | Индикаторы |
|---------|----------|-----------|
| **0 — Не наблюдается** | Нет данных по этой компетенции | Тема не затрагивалась или участник молчал |
| **1 — Осведомлён** | Знает о теме, задаёт базовые вопросы | L1-вопросы, реакции в чате |
| **2 — Применяет** | Использует в своей практике, приводит примеры | L2-вопросы, рассказы о своём опыте |
| **3 — Анализирует** | Видит паттерны, критически оценивает, переносит на другие контексты | L3-вопросы, F-вызовы, кросс-ссылки |
| **4 — Интегрирует** | Объединяет несколько моделей, со-создаёт с ведущим | Со-ведущий, генерация новых кейсов |

**Важно:** уровень — это **наблюдаемый сигнал**, а не оценка человека. Отсутствие сигнала (0) ≠ отсутствие компетенции.

### Шаг 5: ФОРМИРОВАНИЕ ПРОФИЛЯ → Д2

1. Заполнить/обновить все секции карточки
2. Определить **JTBD_Primary** (из таблицы сегментов Cohort Map или уточнить)
3. Рассчитать **SkillFocus** — 3-5 приоритетных навыков на основе:
   - Тематика вопросов → какие S01–S16 релевантны
   - Уровень компетенций → где зона ближайшего развития
   - Сегмент JTBD → приоритеты из матрицы Cohort Map
4. Определить **рекомендуемую AI-роль** (Диагностик / Наставник / Учитель / Коуч) на основе:
   - AI-зрелость → начальная = Наставник, продвинутая = Коуч
   - Стадия компетентности → неосознанная некомпетентность = Диагностик, осознанная = Учитель/Коуч
5. Зафиксировать **риски**:
   - Низкая активность (голос + чат < порога) → риск выпадения
   - Только чат, нет голоса → барьер говорения, breakout поможет
   - Отсутствие на сессиях → проверить вовлечённость
   - Высокая активность + только L1 → риск поверхностного участия
6. Добавить запись в **Хронологию эволюции** с датой и источником
7. **Д2 — Вывести черновик профиля** для ревью

### Шаг 6: АГРЕГАЦИЯ В COHORT MAP

1. Прочитать текущий Cohort Map
2. Обновить данные участника в матрице (если приоритеты сменились)
3. Обновить агрегированные паттерны (например: «3 из 9 перешли с L1 на L2 по Коммуникации»)
4. Добавить запись в секцию «История»

## Диалоговые точки

| # | Когда | Тип | Назначение |
|---|-------|-----|------------|
| **Д1** | После сбора сигналов | Стоп | Подтверждение: какие данные найдены, за какой период, продолжаем? |
| **Д2** | После формирования профиля | Ревью | Черновик карточки, возможность корректировки до сохранения |

**Быстрый режим** («без остановок»): пропустить Д1, показать Д2 и продолжить.

## Шкала классификации вопросов: примеры (504)

| Уровень | Пример вопроса | Классификация |
|---------|---------------|---------------|
| L1/C | «А можно глянуть пример текстового скилла?» | Уточняющий, Meta, Clarification |
| L1/T | «Подробнее про Obsidian — как настроить?» | Уточняющий, Meta, Tool |
| L2/R | «Каждые четыре месяца съезд всего кооператива — это работает для удалёнки?» | Концептуальный, H3 Процессы, Reflection |
| L2/C | «Что знают о личных границах люди, за которыми не бегали с топором?» | Концептуальный, H2 Коммуникация, Clarification |
| L3/F | «А если мы не любим и не уважаем этого человека?» | Интегративный, H2 Коммуникация, Challenge |
| L3/R | «Создать среду, в которой невозможно пойти не туда — а есть конкретный пример?» | Интегративный, H4 Обучение, Reflection |

## Расширяемость (reusable-слой)

Навык спроектирован для 504, но переносим:

| Элемент | 504-specific | Reusable |
|---------|-------------|----------|
| Компетенции | Хоукинз (5) | Любая модель компетенций курса |
| Навыки | S01-S16 | Любая библиотека навыков |
| Сегменты JTBD | A/B/C/D из Cohort Map | Определяются для каждой когорты |
| AI-роли | Диагностик/Наставник/Учитель/Коуч | Сохраняется (4 роли C2) |
| Профиль файл | `504-profile-{Code}` | `{ProjectId}-profile-{Code}` |

При переносе на другой курс: заменить ссылку на Cohort Map, модель компетенций и библиотеку навыков.

## Запреты

- **Ярлыки и оценки личности** — навык оценивает наблюдаемое поведение, не человека
- **Публикация персональных данных** — карточки профилей для ведущего, не для публикации
- **Сравнение участников** — каждый профиль автономен; ранжирование когорты запрещено
- **Диагноз без данных** — если сигналов мало, указать «Недостаточно данных», не додумывать
