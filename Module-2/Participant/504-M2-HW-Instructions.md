## Блок A — Главное задание: value chain + S09

### Job Story (заполните под себя)

> Когда **[ситуация: неясно, где в цепочке теряется ценность / процесс буксует / AI хочется «везде»]**, я провожу **диагностику цепочки создания ценности** с **S09**, чтобы **[увидеть 5–9 шагов, узкое место и 1–2 осмысленные точки для AI или изменения процесса]**.

### Обязательная траектория

1. Откройте [S09 в GDoc](https://docs.google.com/document/d/1xNhSfZBgX-ThuLskqF0_gCENDenxGqqhspEacd7DlBE/edit), перенесите инструкции и **готовый промпт** в свой Claude Project (или аналог).
2. Пройдите **все 6 шагов** оркестратора (границы → элементы → состояние → три слоя AI → приоритизация → антипаттерны).
3. Сохраните итог в формате блока «Выход навыка» из S09 (таблица шагов + точки расширения + антипаттерны) — это ядро ДЗ.

### Три сценария (отметьте, какой выбираете)

| Сценарий | Когда уместен | Фокус |
|----------|---------------|--------|
| **С AI (экзоскелет)** | Есть 45–90 мин на сессию | S09 в диалоге с AI: вопросы → фиксация после каждого шага |
| **Без AI** | Нет доступа к модели | Те же 6 шагов вручную по чеклисту в S09 |
| **На учебном кейсе** | Не хотите свою цепочку | [TechVision](https://docs.google.com/document/d/1XUXruXqnHUzyuMqVGfq5GYonNY47k8ByteFEyU4UcPk/edit) + режим «Учебный кейс» в S09 |

**Ожидаемый артефакт:** заполненная карта value chain + список точек расширения (минимум одна с обоснованием impact/effort).

## Блок A2 — Точечный эксперимент (после карты)

Когда на карте видно **одно узкое место**, выберите **один** навигаторский навык:

| Если узкое место про… | Навык |
|------------------------|--------|
| Фазу команды, конфликт, переход | **S05** |
| Роли, зоны, «кто держит» | **S06** |
| Встречи, ритуалы, ритм | **S07** |
| Передачу задачи, стиль S1–S4 | **S08** |

Проведите **одну** короткую AI-сессию или действие по выбранному навыку **именно на этом шаге** (не «весь процесс сразу»).
