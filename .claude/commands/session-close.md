# Команда: Зафиксируй сессию (session-link — маршрутизация)

**Навык:** [session-link](.cursor/skills/session-link/SKILL.md)

Единая точка входа для фиксации работы: агент **сначала** читает навык и выбирает ветку **START** (начало), **MID** (промежуточный итог) или **CLOSE** (финал). Не завершать сценарий **только** записью в `session-handoff.md` без правок **Objective** и **`YYYY/01-Daily/YYYY-MM-DD.md`**, если пользователь не просил исключительно handoff-текст.

**Жёсткое правило:** канон журнала — **`## Заметки в течение дня`**; **`## Ключевой файл сессии (ctx)`** — опциона��ьный индикатор (**START-7c**).
Локальный `.claude/session-handoff.md` не является журналом дня.

Vault: **504-demo** (`C:\Obs-Vaults\504-demo`). Ежедневны�� заметки: **`YYYY/01-Daily/YYYY-MM-DD.md`**.

## Входные данные ($ARGUMENTS)

- Текст итога / промежуточного состояния, пути к файлам, `ProjectId`, Objective.
- Синонимы: «зафиксируй сессию», «фиксация сессии», «checkpoint», «промежуточный итог», «закрой сессию с записью».

## Обязательные источники

1. `.cursor/skills/session-link/SKILL.md` — разделы «Когда применять», **START**, **MID**, **CLOSE**.
2. `.cursor/skills/session-link/reference.md` — форматы журнала, статусы, ctx, ПЗ.

## Режим

1. Определить ветку по навыку (эвристики или **один** уточняющий вопрос при неоднозначности).
2. **START** — подпротокол START; **MID** — MID; **CLOSE** — CLOSE (в т.ч. CLOSE-4a, CLOSE-7).
3. Один шаг за ответ при уточнениях; запись в vault после подтверждения черновика.

## Итог

Кратко: какая ветка (START / MID / CLOSE), что записано, пути к ПЗ, Objective, файлу дня.

- **START:** Objective `### Сессии` + **START-7b** в `## Заметки в течение дня`; опционально **START-7c** в ctx
- **MID:** ПЗ (промежуточно) + Notes + **MID-6** в журнале; **без** финального статуса
- **CLOSE:** ПЗ + Notes + CLOSE-4a + **CLOSE-7** (смена статуса); опционально CLOSE-8; по умолчанию **CLOSE-5** (session-handoff)
