# Handoff checklist — для участника

> Чеклист «прежде чем нажать «отправить»». Проходить **после** того, как `/mentor-handoff` сгенерировал пакет; **до** фактической отправки получателю.

## Адресат

- [ ] Я знаю, **кому** я отправляю (имя/контакт/канал).
- [ ] Адресат подходит под выбранный уровень обезличивания (mentor / supervisor / therapist / peer / publication).
- [ ] У меня есть **согласие** адресата принять материал (например, ментор знает, что я пришлю).

## Содержимое

- [ ] Я **просмотрел** обезличенный пакет глазами (не пролистал — реально открыл несколько файлов).
- [ ] В обезличенном пакете **нет реальных имён** коллег / руководителей / клиентов.
- [ ] В обезличенном пакете **нет реальных названий компании / продукта** (или они уместны для уровня).
- [ ] Финансовые числа / KPI / показатели — обобщены или удалены (для уровней supervisor / therapist / peer / publication).
- [ ] Цитаты, потенциально вредящие репутации упомянутого человека, — переформулированы или удалены.
- [ ] Файлы из `My-Notes/Private/` **не вошли** в пакет.
- [ ] Mapping `.name-mapping.md` — **остаётся локально**, не уходит вместе с пакетом.

## Юридика и этика

- [ ] У меня нет NDA, который запрещает обсуждение этих материалов.
- [ ] Если есть NDA / corporate policy — материалы не нарушают их (либо я готов взять ответственность).
- [ ] Я понимаю, что после отправки **отозвать назад нельзя** (адресат уже может сохранить).

## Канал отправки

- [ ] Канал безопасен (не публичный мессенджер с открытыми группами, не корпоративная почта).
- [ ] Если по email — адрес проверен; если файлом — формат подходит (PDF — обычно лучший выбор).
- [ ] При желании — попросить адресата подтвердить получение и спросить, нужно ли удалить после прочтения.

## Финальный вопрос

> «Я готов(а), что этот пакет в его текущем виде увидит этот человек?» — да / нет.

Если **нет** — вернуться в `/mentor-handoff` и доработать.
