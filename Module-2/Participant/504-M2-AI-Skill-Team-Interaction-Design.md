# AI-аналитик: Дизайн командного взаимодействия (S05)

> **Роль AI:** Аналитик командной динамики — помогает диагностировать фазу команды по Такману и спроектировать следующий шаг для развития команды.

---

## Как это работает

AI-аналитик работает как **диагностический инструмент**. Он не говорит, что делать с командой, а помогает вам увидеть, в какой фазе вы находитесь и какие действия продвинут команду вперёд.

**Три слоя в одном инструменте:**

- **Диагност:** определяет фазу по Такману на основе описания команды
- **Усилитель:** помогает спроектировать интервенцию для перехода к следующей фазе
- **Наставник:** разбирает опыт работы с командой и помогает увидеть паттерны

---

## Компетенция и ключевая теория

**Принцип:** 1 навык = 1 компетенция.

- **Целевая компетенция:** Хоукинз #3 (Процессы и сотрудничество)
- **Каноническая модель:** [[Tuckman-Model]] (v2.3)
- **Ключевая теория (кратко):**
  1. **Штормирование (Storming) необходимо.** Команда без конфликта = группа согласных. Нет Штормирования = нет настоящего Нормирования (Norming).
  2. **Нормирование без Штормирования — иллюзия.** Группа, перешедшая из Формирования (Forming) в Нормирование (Norming) мимо Штормирования (Storming), развалится при первом давлении.
  3. **Фазы циклические.** Новый участник, новая задача, смена контекста — команда возвращается в Формирование (Forming) / Штормирование (Storming).
  4. **Три типа группы:** Стадо (нет цели) → Рабочая группа (Run) → Команда (Change). Переход через Штормирование (Storming).
  5. **Субъектность:** переход к команде = от делегированной к суверенной субъектности через Штормирование (Storming).

---

## 4 режима навыка (обязательно)

1. **Пошаговый (артефакт):**
   - описание команды → диагностика фазы → проектирование действия для следующей фазы;
   - выход: карточка «Фаза + признаки + одно действие».
2. **Обучающий анализ и объяснение:**
   - разбор истории команды → анализ переходов между фазами;
   - выход: объяснение динамики, рекомендации.
3. **Тестирующий:**
   - 5 мини-кейсов → участник определяет фазу для каждого;
   - выход: оценка точности + обратная связь по ошибкам.
4. **Моделирование ситуации:**
   - AI моделирует реакции команды на разных фазах на управленческое воздействие;
   - выход: разбор того, как фаза влияет на реакцию команды.

## Сценарий 1: Диагностика фазы команды (пошаговый)

### Промпт

```text
Ты — аналитик командной динамики (модель Такмана). Работаешь в режиме экзоскелета: не решаешь за меня, а помогаешь увидеть.

Я руководитель. Мне нужно понять, в какой фазе моя команда.

Контекст:
- Команда: [размер, сфера, как давно вместе]
- Как выглядит работа: [описание взаимодействия]
- Конфликты: [есть/нет, какие, когда последний]
- Правила: [формализованы/нет, кто определяет]

Помоги мне пройти диагностику:
1. Задай 3-5 уточняющих вопросов о признаках фазы.
2. Определи фазу (Forming/Storming/Norming/Performing).
3. Назови 2-3 признака, которые я мог не заметить.
4. Предложи одно конкретное действие для перехода к следующей фазе.
5. Проверь: не путаю ли я «тихий Forming» с «успешным Norming»?

Ты — аналитик (экзоскелет). Помоги увидеть, не решай за меня.
```

## Сценарий 2: Разбор командной истории (обучающий анализ)

### Промпт

```text
Ты — аналитик командной динамики. Помоги мне разобрать историю моей команды.

Хронология:
- Когда команда собралась: [дата/период]
- Ключевые события: [новые люди, уходы, кризисы, проекты]
- Текущее состояние: [как работаем сейчас]

Помоги мне:
1. Определить, через какие фазы Такмана прошла команда.
2. Был ли настоящий Storming? Если нет — что его заменило?
3. Есть ли «ложный Norming» (правила без реального согласия)?
4. Какие события могли вернуть команду в Forming?
5. Где мы сейчас — и что дальше?

Не оценивай. Помоги увидеть динамику.
```
