# Команда: mid-course-diagnostic

Проведи **обязательный mid-course checkpoint** по навыку **ms-participant-profile** — режим 3. Занимает 45–60 минут.

## Входные данные

Контекст: $ARGUMENTS

## Методология

**Канон:** [.cursor/skills/ms-participant-profile/SKILL.md](../../.cursor/skills/ms-participant-profile/SKILL.md), режим 3.
**Шаблон отчёта:** [.cursor/skills/ms-participant-profile/templates/mid-course-diagnostic.template.md](../../.cursor/skills/ms-participant-profile/templates/mid-course-diagnostic.template.md)

## Порядок выполнения

1. Прочитай `ms-profile.md v1` / последнюю версию + `goals-*.md` + все HW-Protocol за пройденные модули + daily-notes.
2. **Шаг 1:** сравнение деклараций и фактов (две колонки).
3. **Шаг 2:** скрытая мотивация — один вопрос участнику, сравнение с декларацией.
4. **Шаг 3:** соответствие оставшейся программы — проверить, что M3, M4 работают на новый или старый вектор.
5. **Шаг 4 (обязательно):** контр-диагностика — 3 альтернативных объяснения сдвига, какую нельзя опровергнуть, что меняется.
6. **Шаг 5:** участник делает явный выбор: Continue / Adjust / Pivot.
7. **Шаг 6:** записать отчёт `mid-course-diagnostic-YYYY-MM-DD.md` по шаблону, обновить `ms-profile.md` (актуальные цели).

## Итог

Три артефакта: отчёт, обновлённый профиль, явное решение «как идём дальше на оставшиеся модули».

## Важно

**Не пропускать контр-диагностику** даже при запросе участника «давай быстрее». Это защита от селективности восприятия — см. раздел «Жёсткие правила» в SKILL.md.
